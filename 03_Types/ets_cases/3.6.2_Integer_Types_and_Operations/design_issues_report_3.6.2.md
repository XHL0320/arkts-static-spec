# 3.6.2 Integer Types and Operations - ArkTS 设计问题发现报告

**报告日期：** 2026-06-11
**测试用例数：** 21（10 compile-pass + 3 compile-fail + 8 runtime）
**目的：** 通过用例执行 + 跨语言对比，识别 ArkTS 整数运算设计

---

## 一、本章节首次发现的设计问题

### 问题 Q：`as` 表达式弃用于数值类型转换 ⭐⭐ MEDIUM

**用例：** TYP_03_06_02_014（修复前）

**实际行为：**
```typescript
let i: int = 100
let l: long = i as long
// ❌ ESE123811: 'As' expression for cast is deprecated for numeric types.
//    Use explicit conversion function Int.toLong(...) instead.

let l2: long = i.toLong()   // ✅ 必须用方法
```

**对比：**
| 语言 | 数值类型转换语法 |
|------|----------------|
| ArkTS | `i.toLong()` 方法（仅此一种）|
| Java | `(long) i` 强制类型转换 |
| Swift | `Int64(i)` 构造器 |

**spec 描述：** §3.6.2 未明确禁止 `as`，编译器报错信息提到 stdlib 推荐方法

**评价：**
- 这是 **ArkTS 独特设计**，与 Java/Swift 都不同
- `as` 在 ArkTS 其他场景（reference type cast）仍可用
- 但对数值类型禁用 `as`，造成同一关键字在不同上下文行为不一致
- 强制使用 stdlib 方法（`toInt()`/`toLong()`/`toDouble()` 等）增加了一致性，但学习成本高

**建议：**
1. spec §3.6.2 应明确说明数值转换必须用 stdlib 方法
2. 编译错误信息已经清晰，但 spec 文档应补充
3. 考虑同时支持 `as` 和 `toXxx()`（兼容性）或明确文档化此限制

---

### 问题 R：`**` 整数幂运算返回 bigint 的设计意图 ⭐ LOW

**spec §3.6.2 描述：**
> Exponentiation operator '**' as a variant that produces a value of type bigint

**用例：** TYP_03_06_02_010

**对比：**
| 语言 | 整数幂运算 |
|------|----------|
| ArkTS | `**` 返回 bigint |
| Java | 无 `**` 运算符（用 Math.pow）|
| Swift | 无 `**` 运算符（用 pow()）|

**评价：**
- ArkTS 是三者中唯一有 `**` 的整数运算符
- 返回 bigint 避免了 `2 ** 30` 这种容易溢出 int 的场景
- 但用户必须熟悉 bigint，可能感到意外

**建议：**
- spec 已明确说明，无需修改
- 文档可加入"为何选择 bigint"的解释

3. 考虑借鉴 Swift `&+` `&-` 显式溢出运算符（解决整数溢出安全问题，复用 3.1 TYP-001 建议）
