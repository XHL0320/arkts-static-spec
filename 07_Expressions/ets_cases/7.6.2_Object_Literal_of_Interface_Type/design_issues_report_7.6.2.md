# 7.6.2 Object Literal of Interface Type — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: 接口对象字面量语法（ArkTS 独有） ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | 全部 compile-pass 用例 |
| **实测结果** | ArkTS 原生支持接口对象字面量语法，隐式创建匿名类 |
| **错误信息** | 无 |

**描述**：ArkTS 是唯一原生支持接口对象字面量语法的语言，隐式创建匿名类。Java 使用匿名内部类，Swift 需预定义实现类。

**跨语言对比**：

| 语言 | 接口实现方式 | 说明 |
|------|-------------|------|
| ArkTS | `let b: Person = {name: "Bob", age: 25}` | 原生隐式匿名类 |
| Java | `Person p = new Person() { ... }` | 匿名内部类（冗长） |
| Swift | `let p = PersonImpl()` | 需预定义实现类 |

**分类**：语言特性差异

**建议**：符合各自语言设计，无需调整。

### ID-02: 属性读写形式的细粒度控制 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_06_02_009~012 四个属性形式用例 |
| **实测结果** | ArkTS 支持 setter-only/getter-only/readonly/regular 四种属性形式 |
| **错误信息** | 无 |

**描述**：ArkTS 接口支持 setter-only/getter-only/readonly/regular 四种属性形式，在对象字面量中编译时检查。Java/Swift 需通过方法签名或 {get}/{set} 约束。

**跨语言对比**：

| 形式 | ArkTS | Java | Swift |
|------|-------|------|-------|
| setter-only | ✅ 可写不可读 | ❌ 无等价 | ❌ 属性至少需 get |
| getter-only | ✅ 可读不可写 | ❌ 方法签名不同 | ✅ `{ get }` |
| readonly | ✅ 可读不可写 | `final` 字段 | `let` |
| regular | ✅ 可读写 | ✅ getter+setter | ✅ get+set |

**分类**：类型系统设计差异

**建议**：符合 ArkTS 设计，无需调整。

### ID-03: 新方法检查 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_06_02_015_FAIL_NEW_METHOD |
| **实测结果** | 编译时错误（期望行为） |
| **错误信息** | 编译时错误 |

**描述**：ArkTS 禁止对象字面量引入新方法（编译时错误），Java/Swift 允许匿名类/实现类添加额外方法。

**跨语言对比**：

| 语言 | `const i: I = { foo(): void {} }`（`interface I {}`） |
|------|------------------------------------------------------|
| ArkTS | ❌ 编译时错误（严格检查） |
| Java | ✅ 匿名类可加额外方法 |
| Swift | ✅ 协议实现可加额外方法 |

**分类**：编译时检查策略差异

**建议**：符合 ArkTS 设计，无需调整。

### ID-04: 可选属性默认值 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_06_02_002_PASS_OPTIONAL_SKIP |
| **实测结果** | 可选属性跳过值为 undefined |
| **错误信息** | 无 |

**描述**：三语言对可选属性的处理语义等价：ArkTS `undefined`、Java `null`、Swift `nil`。

**跨语言对比**：

| 语言 | 可选属性值 |
|------|-----------|
| ArkTS | `undefined` |
| Java | `null` |
| Swift | `nil` |

**分类**：语法机制差异

**建议**：语义等价，无需调整。
