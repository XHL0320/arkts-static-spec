# 7.32.2 Compound Assignment Operators - 测试设计思维导图

## 概述

验证 ArkTS 中复合赋值运算符（`+=` `-=` `*=` `/=` `%=` `<<=` `>>=` `>>>=` `&=` `|=` `^=` `??=`）的正确性，包括每种运算符在不同数值类型上的编译通过、与 `=` 的等价语义、三种 lhsExpression 形式（变量/字段访问、数组索引、记录索引）、空值合并赋值 `??=` 的特殊语义以及运行时求值顺序和越界检查。

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 算术复合赋值 | `+=` `-=` `*=` `/=` `%=` 在 int/long/float/double 类型上 |
| 002 | 移位/位运算复合赋值 | `<<=` `>>=` `>>>=` `&=` `|=` `^=` 在 int/long/boolean 类型上 |
| 003 | 字符串拼接复合赋值 | `string += string` 和 `string += int` |
| 004 | 字段/数组/记录复合赋值 | `obj.field += rhs`、`arr[idx] += rhs`、`rec[key] += rhs` |
| 005 | 空值合并赋值 ??= | nullable 类型上 ??= 的正常用法 |
| 006 | 规范示例验证 | 变量/字段/数组/记录复合赋值各运算符 |

### compile-fail

| # | 测试点 | 说明 |
|---|--------|------|
| 007 | 运算符类型不匹配 | `string -=`、`boolean +=`、`int += string` 等 10 种不兼容场景 |
| 008 | ??= 在非 nullable 类型上 | 非空值类型使用空值合并赋值报错 |

### runtime（验证实际计算值符合复合赋值语义）

| # | 测试点 | 预期值 |
|---|--------|--------|
| 009 | 运行时语义验证 | 20+ 断言：算术/移位/位运算/字符串/字段/数组/记录复合赋值值正确 |
| 010 | 数组复合越界 RangeError | arr[-1] += 99 → RangeError；arr[5] += 5 (len=3) → RangeError |
| 011 | ??= 运行时行为 | nullable 变量 ??= 赋值/跳过的运行时行为 |
