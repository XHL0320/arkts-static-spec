# 7.32.3 Left Hand Side Expressions - 测试设计思维导图

## 概述

验证 ArkTS 中左值表达式（Left-Hand-Side Expression）的正确性。左值表达式是赋值（包括简单赋值 `=`、复合赋值 `+=` 等、自增自减 `++`/`--`）的目标表达式。Spec 定义了哪些表达式可以作为左值、哪些会导致编译时错误。

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 变量作为左值 | int/long/float/double/string/boolean 六种类型变量赋值 |
| 002 | 参数作为左值 | 函数/方法参数（非 this）重新赋值 |
| 003 | 字段访问作为左值 | 公有字段赋值、嵌套字段赋值、链式字段赋值 |
| 004 | 数组索引作为左值 | 常量索引、变量索引、二维数组索引 |
| 005 | 记录索引作为左值 | 字符串字面量 key、变量 key 赋值 |
| 006 | 复合赋值/自增自减在左值上 | 变量/字段复合赋值和自增自减 |
| 007 | 嵌套左值表达式 | 字段+数组组合的嵌套左值 |

### compile-fail

| # | 测试点 | 说明 |
|---|--------|------|
| 008 | 字面量作为左值 | 数值/字符串/布尔字面量在赋值左侧 |
| 009 | 方法调用作为左值 | 方法/函数调用在赋值左侧 |
| 010 | 算术表达式作为左值 | 算术运算结果在赋值左侧 |
| 011 | 可选链 `?.` 作为左值 | 可选链字段/方法/数组在赋值左侧 |
| 012 | 只读实体作为左值 | const 常量、readonly 字段/数组元素 |
| 013 | Lambda 作为左值 | Lambda 表达式在赋值左侧 |
| 014 | 数组字面量作为左值 | 数组字面量在赋值左侧 |
| 015 | 对象字面量作为左值 | 对象字面量在赋值左侧 |
| 016 | 三元表达式作为左值 | 三元条件表达式在赋值左侧 |
| 017 | Nullish Coalescing 作为左值 | null 合并表达式在赋值左侧 |
| 018 | instanceof 作为左值 | instanceof 表达式在赋值左侧 |
| 019 | typeof 作为左值 | typeof 表达式在赋值左侧 |
| 020 | new 表达式作为左值 | new 表达式在赋值左侧 |
| 021 | readonly 字段作为左值 | readonly 字段在赋值左侧 |

### runtime（验证左值赋值语义正确性）

| # | 测试点 | 预期值 |
|---|--------|--------|
| 022 | 运行时赋值语义验证 | 17 断言：变量/参数/字段/数组/记录/复合赋值均正确 |

## 文件命名规范

`EXP_07_32_03_YYY_{CATEGORY}_{DESCRIPTION}.ets`

- YYY: 001 起连续编号
- CATEGORY: PASS / FAIL / RUNTIME
- DESCRIPTION: 大写+下划线描述

## 注释模板要求

每个文件必须包含 5 个 JSDoc tag：@id, @expect, @section, @design, @note
