# 7.1.1 Operator Precedence — 三语言对比报告

## 1. 概览

运算符优先级定义了复合表达式中运算符的计算顺序。本节对比 ArkTS / Java / Swift 三语言的运算符优先级规则和结合性。

## 2. 章节对应关系

| 概念 | ArkTS | Java (JLS SE21) | Swift (5.10) |
|------|-------|----------------|--------------|
| 运算符优先级表 | 7.1.1 Operator Precedence | §15.2 Operator Precedence | precedencegroup 声明 |
| 结合性 | left-to-right / right-to-left | left-to-right / right-to-left | left / right / none |
| 分组 | `()` | `()` | `()` |
| 成员访问 | `.`, `?.` | `.` | `.`, `?.` |
| 幂运算 | `**` | `Math.pow()` | `pow()` |
| 移位 | `<<` `>>` `>>>` | `<<` `>>` `>>>` | `<<` `>>` `&>>` |
| 赋值 | `=` `+=` 等 | `=` `+=` 等 | `=` `+=` 等（不返回） |

## 3. 关键差异矩阵

| 优先级特征 | ArkTS | Java | Swift |
|-----------|-------|------|-------|
| 移位与算术优先级 | 算术 > 移位 | 算术 > 移位 | 移位 > 算术 |
| 赋值返回值 | 返回赋值 | 返回赋值 | Void（不返回） |
| 自增自减 `++`/`--` | ✅ | ✅ | ❌（3.0+ 移除） |
| 幂运算符 `**` | ✅ 内置 | ❌ 用 Math.pow | ❌ 用 pow |
| 三元条件类型 | 需 boolean | 需 boolean | 需 BooleanExpression |
| 比较运算符链 | 允许 | 允许 | 不允许（non-associative） |
| `?.` 可选链 | ✅ | ❌ | ✅ |
| 一元负号与 `**` 优先级 | 一元 > `**` | n/a | n/a |

## 4. 用例对照

### 4.1 括号分组优先级
```
ArkTS:  (2 + 3) * 4  → 20,  2 + 3 * 4  → 14
Java:   (2 + 3) * 4  → 20,  2 + 3 * 4  → 14
Swift:  (2 + 3) * 4  → 20,  2 + 3 * 4  → 14
```
✅ 一致

### 4.2 移位与算术混合
```
ArkTS:  1 + 2 << 3 - 1  → 12  (加法 > 移位)
Java:   1 + 2 << 3 - 1  → 12  (加法 > 移位)
Swift:  1 + 2 << 3 - 1  → 16  (移位 > 加法)
```
⚠️ Swift 的移位优先级高于加减，与 ArkTS/Java 相反

### 4.3 一元负号与幂运算
```
ArkTS:  (-3) ** 2    → 9   (一元高于 **，但编译器要求括号)
Java:   -Math.pow(3, 2) → -9  (方法调用高于一元)
Swift:  -pow(3.0, 2.0) → -9  (函数调用高于一元)
```
⚠️ ArkTS 规范显示一元运算符高于幂运算，但编译器要求括号消除歧义

### 4.4 赋值链
```
ArkTS:  a = b = c = 5  → a=5,b=5,c=5  ✅
Java:   a = b = c = 5  → a=5,b=5,c=5  ✅
Swift:  不支持链式赋值（赋值返回 Void） ❌
```
⚠️ Swift 赋值返回 Void，需分开赋值

| 维度 | ArkTS | Java | Swift |

## 7. 核心结论

1. **ArkTS 运算符优先级与 Java 高度一致**，所有 21 个优先级层级的排序与 Java 相同。
2. **Swift 在以下方面与 ArkTS/Java 不同**：移位优先级高于算术（Swift 设计选择）、赋值不返回（安全性设计）、移除了自增/自减运算符。
3. **ArkTS 特有的运算符**包括 `**`（幂运算，返回 double）和 `?.`（可选链访问），这些 Java 没有直接等价。
4. **一元负号与幂运算**在 ArkTS 中需要括号消除语法歧义，与 Java/Swift 的解析方式不同。
5. **比较运算符链**在 Swift 中是非结合的，需要括号明确分组，而 ArkTS/Java 允许自然链接。

## 8. ArkTS 设计建议

1. **当前设计合理**：优先级层级与 Java 高度一致，降低了 Java 开发者的学习成本。
2. **幂运算的返回值类型**：当前 `**` 返回 `double`，在处理整数幂运算时需要显式转换。可考虑提供整数幂运算的重载。
