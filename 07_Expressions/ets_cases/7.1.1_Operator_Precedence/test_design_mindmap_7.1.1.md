# 7.1.1 Operator Precedence - 测试设计思维导图

## 概述

运算符优先级定义了复合表达式中不同运算符的计算顺序。结合性（associativity）定义了相同优先级运算符的计算方向（左到右或右到左）。括号 `()` 可改变默认优先级。

## 优先级层级（从高到低）

| 层级 | 运算符 | 结合性 | 说明 |
|------|--------|--------|------|
| 1 | `()` | n/a | 分组 |
| 2 | `.`, `?.` | left-to-right | 成员访问和链式调用 |
| 3 | `[]`, `.`, `()`, `new` | n/a | 访问和调用 |
| 4 | `++`, `--` (postfix) | n/a | 后置递增/递减 |
| 5 | `!` (postfix, ensure-not-nullish) | n/a | 后置非空断言 |
| 6 | `++`, `--`, `+`, `-`, `!`, `~`, `typeof`, `await` (prefix) | n/a | 前置一元运算符 |
| 7 | `**` | right-to-left | 幂运算 |
| 8 | `*`, `/`, `%` | left-to-right | 乘法 |
| 9 | `+`, `-` | left-to-right | 加法 |
| 10 | `as` | left-to-right | 类型转换 |
| 11 | `<<`, `>>`, `>>>` | left-to-right | 移位 |
| 12 | `<`, `>`, `<=`, `>=`, `instanceof` | left-to-right | 关系 |
| 13 | `==`, `!=` | left-to-right | 相等 |
| 14 | `&` | left-to-right | 按位与 |
| 15 | `^` | left-to-right | 按位异或 |
| 16 | `\|` | left-to-right | 按位或 |
| 17 | `&&` | left-to-right | 逻辑与 |
| 18 | `\|\|` | left-to-right | 逻辑或 |
| 19 | `??` | left-to-right | null 合并 |
| 20 | `? :` | right-to-left | 三元条件 |
| 21 | `=`, `+=`, `-=`, `%=`, `*=`, `/=`, `&=`, `^=`, `\|=`, `<<=`, `>>=`, `>>>=` | right-to-left | 赋值 |

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 分组括号优先级 | `(2 + 3) * 4` 与 `2 + 3 * 4` 应产生不同结果 |
| 002 | 乘法优先于加法 | `2 + 3 * 4` = 14 |
| 003 | 加法优先于移位 | `1 + 2 << 3` 先加再移位 |
| 004 | 移位优先于关系 | `4 << 2 > 10` 先移位再比较 |
| 005 | 关系优先于相等 | `3 < 5 == true` 合法编译 |
| 006 | 按位与优先于异或优先于或 | `1 \| 2 & 3 ^ 4` 按 `1 \| (2 & (3 ^ 4))` 计算 |
| 007 | 逻辑与优先于逻辑或 | `true \|\| false && false` 等价 `true \|\| (false && false)` |
| 008 | 赋值右结合链 | `a = b = 5` 从右向左赋值 |
| 009 | 幂运算右结合 | `2 ** 3 ** 2` = 2^(3^2) = 512 |
| 010 | 三元条件右结合 | `a ? b : c ? d : e` 等价 `a ? b : (c ? d : e)` |
| 011 | 一元负号与幂运算 | `-3 ** 2` = -(3^2) = -9 |
| 012 | 类型转换优先级 | `x as int + 1` vs `(x as int) + 1` 不同解析 |
| 013 | 成员访问优先于算术 | 对象字段在算术前访问 |
| 014 | null 合并优先于三元 | `a ?? b ? c : d` 等价 `(a ?? b) ? c : d` |
| 015 | 复合赋值与算术 | `a += b * c` 先算 `b * c` 再复合赋值 |
| 016 | 混合运算符链 | 多个不同优先级运算符组合 |
| 017 | 后置递增与算术 | `a++ + b` 先取 a 值再加 b，然后 a 自增 |
| 018 | 前置递增与幂运算 | `++a ** 2` 先递增再求幂 |
| 019 | 字符串拼接与加法 | `"a" + 1 + 2` vs `"a" + (1 + 2)` |
| 020 | 按位运算符与移位混合 | `1 << 2 + 3 & 4` 按优先级计算 |

### compile-fail

| # | 测试点 | 说明 |
|---|--------|------|
| 021 | 赋值给非左值 | `a + b = c` 加法优先，赋值给非左值 |
| 022 | 连续后置递增 | `a++ ++` 后置 ++ 不能重复应用 |
| 023 | postfix `!` 后跟数字 | 表达式 `a! + 1` 的解析歧义（需验证 compile-time error） |
| 024 | 赋值与关系比较混淆 | `if (a = b)` 赋值返回非 boolean（如有此限制） |

### runtime（验证实际计算值符合优先级规则）

| # | 测试点 | 预期值 |
|---|--------|--------|
| 025 | 括号覆盖优先级 | `(2 + 3) * 4` = 20, `2 + 3 * 4` = 14 |
| 026 | 算术混合优先级 | `1 + 2 * 3 - 4 / 2` = 1 + 6 - 2 = 5 |
| 027 | 移位与算术混合 | `1 + 2 << 3 - 1` = `(1+2) << (3-1)` = 3 << 2 = 12 |
| 028 | 关系与相等链 | `1 < 2 == true` = true |
| 029 | 位运算优先级链 | `1 \| 2 & 3 ^ 4` 按位计算验证 |
| 030 | 逻辑运算符短路与优先级 | `true \|\| false && false` = true |
| 031 | 赋值右结合 | `a = b = 3` 后 a=3, b=3 |
| 032 | 幂运算右结合 | `2 ** 3 ** 2` = 512 |
| 033 | 三元右结合 | 嵌套三元按右结合解析 |
| 034 | null 合并优先级 | `null ?? 1 + 2` = 3 |
| 035 | 混合运算整体验证 | 含多级优先级的复杂表达式 |

- DESCRIPTION: 大写+下划线描述
