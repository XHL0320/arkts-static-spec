# 7.4.2 Method Reference - 测试设计思维导图

## 概述

方法引用（Method Reference）指向类的静态或实例方法。方法引用的类型从方法签名推导得出。实例方法引用被绑定到调用时使用的实例对象。ArkTS 支持静态方法引用、实例方法引用、泛型方法引用（需显式类型参数），以及对重载方法名的引用限制。

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 静态方法引用 | `C.foo` 方式引用类的静态方法，类型由方法签名推导 |
| 002 | 实例方法引用 | `new C().bar` 方式引用实例方法，类型由方法签名推导 |
| 003 | 实例绑定引用 | `c1.method` 绑定到 `c1`，`c2.method` 绑定到 `c2`，不同实例的状态隔离 |
| 004 | 泛型方法引用 | 泛型方法引用带显式类型参数编译通过 |
| 005 | 显式重载单方法引用 | 显式重载声明中单个方法名可作为引用 |

### compile-fail（验证编译时错误）

| # | 测试点 | 说明 |
|---|--------|------|
| 006 | 泛型方法引用无类型参数 | 泛型方法引用不带显式类型参数，应报编译时错误 |
| 007 | 隐式重载方法名引用 | 类中隐式重载方法名作为引用，应报编译时错误 |
| 008 | 显式重载方法名引用 | `overload foo { foo1, foo2 }` 中 `c.foo` 作为引用，应报编译时错误 |

### runtime（验证实际运行结果正确）

| # | 测试点 | 预期值 |
|---|--------|--------|
| 009 | 实例绑定运行时隔离 | `m1()=42`, `m2()=123` 实例绑定隔离性验证 |
| 010 | 静态方法引用调用 | `Calc.add(10,20)==30` |
| 011 | 泛型方法引用调用 | `identity<int>(99)==99` |

## 文件命名规范

`EXP_07_04_02_YYY_{CATEGORY}_{DESCRIPTION}.ets`

- YYY: 001 起连续编号
- CATEGORY: PASS / FAIL / RUNTIME
- DESCRIPTION: 大写+下划线描述

## 注释模板要求

每个文件必须包含 5 个 JSDoc tag：@id, @expect, @section, @design, @note
