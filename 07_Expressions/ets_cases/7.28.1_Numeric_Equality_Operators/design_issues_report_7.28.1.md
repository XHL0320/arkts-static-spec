# 7.28.1 Numeric Equality Operators — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: Object 类型与数值类型等值比较 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_28_01_015_FAIL_OBJECT_VS_NUMERIC.ets |
| **实测结果** | 编译通过（无错误） |
| **错误信息** | 无错误 |

**描述**：Spec 7.28.1 要求"每个操作数必须可转换为数值类型或 bigint 类型，否则编译时错误"。代码 `let o: Object = new Object(); let r = o == 5;` 中 Object 不可转换为数值类型或 bigint 类型，应产生编译时错误，但实际编译通过。可能原因是编译器实现了 Object 到数值类型的某种隐式转换（或 == 对所有引用类型通用）。

**跨语言对比**：

| 语言 | Object == int | 行为 |
|------|:------------:|------|
| ArkTS | 允许（D 类） | 编译通过 |
| Java | 编译时错误 | 不允许不同类型比较 |
| Swift | 编译时错误 | 不允许不同类型比较 |

**分类**：D 类（Spec 与实现不一致）

**建议**：评估是否为有意设计，若允许则需更新 Spec。

### ID-02: 枚举类型与数值类型等值比较 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_28_01_016_FAIL_ENUM_VS_NUMERIC.ets |
| **实测结果** | 编译通过（无错误） |
| **错误信息** | 无错误 |

**描述**：Spec 7.28.1 要求枚举不可转换为数值类型或 bigint 类型，代码 `Color.Red == 0`（int 基类型枚举）应产生编译时错误，但实际编译通过。编译器实现了枚举到基类型的隐式转换（enum → int），使 `Color.Red == 0` 退化为 `0 == 0` 的 int 比较。与 7.27.6 的 D 类相同模式。

**跨语言对比**：

| 语言 | enum == int | 行为 |
|------|:-----------:|------|
| ArkTS | 允许（D 类） | 编译通过 |
| Java | 编译时错误 | 不允许不同类型比较 |
| Swift | 编译时错误 | 不允许不同类型比较 |

**分类**：D 类（Spec 与实现不一致）

**建议**：与 7.27.6 的枚举关系运算 D 类问题属于同一模式，建议统一处理。

### ID-03: null/undefined 与数值类型等值比较 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_28_01_017_FAIL_NULLISH_WITH_NUMERIC.ets |
| **实测结果** | 编译通过（无错误） |
| **错误信息** | 无错误 |

**描述**：Spec 7.28.1 要求 null/undefined 不可转换为数值类型或 bigint 类型，代码 `null == 5` / `undefined == 5` 应产生编译时错误，但实际编译通过。可能原因是 null/undefined 的等值比较由 7.28.3 Extended Equality 处理，不在 7.28.1 的数值等值路径中。

**跨语言对比**：

| 语言 | null == int | 行为 |
|------|:-----------:|------|
| ArkTS | 允许（D 类） | 编译通过 |
| Java | 编译时错误 | 不允许不同类型比较 |
| Swift | 编译时错误 | 不允许不同类型比较 |

**分类**：D 类（Spec 与实现不一致）

**建议**：评估 null/undefined 与数值的比较是否应归入 7.28.3 Extended Equality 而非 7.28.1。

**D 类分析**：三个 D 类问题均出现在非数值类型与数值类型的 `==` 比较中。Spec 7.28.1 要求"每个操作数必须可转换为数值类型或 bigint 类型，否则编译时错误"，但实现允许：1. Object 类型通过 `==` 与数值比较；2. 枚举类型通过隐式基类型转换与数值比较（与 7.27.6 相同模式）；3. null/undefined 通过 extended equality 与数值比较（可能归入 7.28.3）。

### ID-05: bigint 支持差异 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | N/A（设计差异分析） |
| **实测结果** | ArkTS 是三种语言中唯一原生支持 bigint 等值比较的语言 |
| **错误信息** | N/A |

**描述**：ArkTS 原生支持 bigint 类型及其等值比较（`==`/`!=`/`===`/`!==`），bigint 与 numeric 比较结果为 false。Java 没有 bigint 类型，使用 BigInteger 类需通过 `equals()` 方法比较。Swift 无 bigint 类型。

**跨语言对比**：

| 语言 | bigint 类型 | bigint 等值 | bigint vs numeric |
|------|:----------:|:-----------:|:-----------------:|
| ArkTS | bigint | 支持 `==`/`!=`/`===`/`!==` | 结果为 false |
| Java | BigInteger 类 | 需 `equals()` | N/A |
| Swift | 无 | N/A | N/A |

**分类**：跨语言设计差异

### ID-06: ===/!== 运算符差异 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | N/A（设计差异分析） |
| **实测结果** | ArkTS 中 === 与 == 语义相同，Java/Swift 无此对应 |
| **错误信息** | N/A |

**描述**：ArkTS 对数值类型使用 `===`/`!==` 与 `==`/`!=` 语义相同。Java 无 `===` 运算符。Swift 的 `===` 仅用于引用相等，不适用于数值类型。

**跨语言对比**：

| 语言 | 数值 ===/!== |
|------|:-----------:|
| ArkTS | 与 ==/!= 语义相同 |
| Java | 无此运算符 |
| Swift | === 仅用于引用相等 |

**分类**：跨语言设计差异

### ID-07: long 字面量语法差异 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | N/A（设计差异分析） |
| **实测结果** | ArkTS 无需 L 后缀，Java 需要 5L 后缀 |
| **错误信息** | N/A |

**描述**：ArkTS 中 long 类型赋值无需特殊后缀，直接使用 `let l: long = 5`。Java 需要使用 `5L` 后缀。Swift 通过 Int64 类型推断。

**跨语言对比**：

| 语言 | long 字面量 |
|------|:----------:|
| ArkTS | 无 `L` 后缀，直接赋值 `let l: long = 5` |
| Java | `5L` 后缀 |
| Swift | Int64 类型推断 |

**分类**：跨语言设计差异
