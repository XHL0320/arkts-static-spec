### ID-02: 索引语法差异 ⭐

| 字段 | 值 |
|------|-----|
| **问题描述** | ArkTS 使用 `s[i]` 统一数组和字符串索引语法；Java 使用 `charAt()`；Swift 使用 `String.Index` |
| **分类** | 符合 ArkTS spec 的语言设计差异 |
| **严重性** | ⭐（低）—— 语法差异，不影响功能正确性 |

**跨语言对比**：

| 语言 | 代码 | 说明 |
|------|------|------|
| ArkTS | `s[i]` | 直接 `[]` 语法，类似数组索引 |
| Java | `s.charAt(i)` | 方法调用，非 `[]` 操作符 |
| Swift | `s[s.index(startIndex, offsetBy: i)]` | 通过 `String.Index` 间接索引 |

---

### ID-03: 返回类型差异 ⭐

| 字段 | 值 |
|------|-----|
| **问题描述** | ArkTS 字符串索引返回 `string` 类型（单字符子串），Java 返回 `char`，Swift 返回 `Character` |
| **分类** | 符合 ArkTS spec 的语言设计差异 |

**跨语言对比**：

| 语言 | 返回类型 | 示例 |
|------|---------|------|
| ArkTS | `string` | `"abc"[1]` → `"b"` |
| Java | `char` | `"abc".charAt(1)` → `'b'` |
| Swift | `Character` | `"abc"[index]` → `"b"` |

**ArkTS 设计特点**：返回 `string` 而非 `char`，与其他语言不同。`char` 在 ArkTS 中虽然是关键字，但字符串索引直接返回子串（`string` 类型）。

### ID-04: 不可变性检查 — 三语言一致

| 字段 | 值 |
|------|-----|
| **问题描述** | 三语言都在编译时阻止通过索引修改字符串元素 |
| **分类** | 三语言一致 |
| **结论** | 符合各语言 spec |

**跨语言对比**：

| 语言 | 代码 | 行为 |
|------|------|------|
| ArkTS | `s[1] = "d"` | ❌ 编译错误 |
| Java | `s.charAt(1) = 'd'` | ❌ 编译错误（charAt 不返回可赋值的 lvalue） |
| Swift | `s[someIndex] = "d"` | ❌ 编译错误（subscript is get-only） |

---

### ID-05: 越界运行时行为差异 ⭐

| 字段 | 值 |
|------|-----|
| **问题描述** | 三语言字符串索引越界均抛出运行时异常，但异常类型不同 |
| **分类** | 符合 ArkTS spec 的语言设计差异 |

**跨语言对比**：

| 语言 | 越界行为 | 可捕获 |
|------|---------|--------|
| ArkTS | `RangeError` | ✅ `try/catch` |
| Java | `StringIndexOutOfBoundsException` | ✅ `try/catch` |
| Swift | `fatal error` | ❌ 不可捕获（但支持 `limitedBy:` 安全方式） |
