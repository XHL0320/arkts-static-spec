# 7.4.1 Function Reference - 测试设计思维导图

## 概述

函数引用（Function Reference）指向已声明或导入的函数。函数引用的类型从函数签名推导得出，可赋值给变量、作为参数传递或直接调用。ArkTS 支持直接函数名引用、泛型函数引用（需显式类型参数）、以及对重载函数引用的严格编译时检查。

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 基本函数引用 | 直接引用已声明函数，类型由函数签名自动推导 |
| 002 | 函数引用类型推导 | 多签名函数引用类型推导正确 |
| 003 | 通过引用调用函数 | 将函数引用赋值给变量后，通过变量调用原函数 |
| 004 | 泛型函数引用 | 泛型函数引用必须携带显式类型参数，如 `gen<string>` |
| 005 | 显式重载单函数引用 | 显式重载声明中单个函数名可作为引用 |

### compile-fail（验证编译时错误）

| # | 测试点 | 说明 |
|---|--------|------|
| 006 | 泛型无类型参数引用 | 泛型函数引用不带显式类型参数，应报编译时错误 |
| 007 | 隐式重载函数名引用 | 同名的多个重载函数（隐式重载）名称作为引用，应报编译时错误 |
| 008 | 显式重载名称引用 | `overload foo { foo1, foo2 }` 中 `foo` 名称作为引用，应报编译时错误 |

### runtime（验证实际运行结果正确）

| # | 测试点 | 预期值 |
|---|--------|--------|
| 009 | 函数引用调用返回值 | `func(3,5) == 8` |
| 010 | 泛型引用调用 | `identity<int>(42) == 42` |

## 文件命名规范

`EXP_07_04_01_YYY_{CATEGORY}_{DESCRIPTION}.ets`

- YYY: 001 起连续编号
- CATEGORY: PASS / FAIL / RUNTIME
- DESCRIPTION: 大写+下划线描述

## 注释模板要求

每个文件必须包含 5 个 JSDoc tag：@id, @expect, @section, @design, @note
