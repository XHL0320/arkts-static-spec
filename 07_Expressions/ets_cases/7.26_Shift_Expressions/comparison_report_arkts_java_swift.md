# 7.26 Shift Expressions — 三语言对比报告

## 1. 概览

移位表达式定义 `<<`（左移）、`>>`（带符号右移）、`>>>`（无符号右移）三种运算。

| 语言 | 移位运算符 | 溢出处理 | 无符号右移 |
|------|-----------|---------|-----------|
| ArkTS | `<<`, `>>`, `>>>` | 二进制补码截断（静默） | 原生支持 |
| Java | `<<`, `>>`, `>>>` | 二进制补码截断（静默） | 原生支持 |
| Swift | `&<<`, `>>`, （无 `>>>`） | 默认溢出检测，需 `&<<` | 需自定义实现 |

## 2. 章节对应关系

| 功能点 | ArkTS 7.26 | Java JLS §15.19 | Swift 表达式 |
|--------|-----------|-----------------|-------------|
| 左移 | `<<` | `<<` | `&<<`（溢出）或 `<<`（默认检测）|
| 带符号右移 | `>>` | `>>` | `>>` |
| 无符号右移 | `>>>` | `>>>` | 无原生，需 UInt 转换 |
| bigint 移位 | `<<`, `>>` | N/A | N/A |
| int 距离掩码 | 0x1f（5位） | 0x1f（5位） | 0x1f（Int32）/ 0x3f（Int64） |
| long 距离掩码 | 0x3f（6位） | 0x3f（6位） | 0x3f（Int64） |

## 3. 关键差异矩阵

| 差异点 | ArkTS | Java | Swift |
|--------|:-----:|:----:|:-----:|
| 溢出处理方式 | 静默截断 | 静默截断 | 默认异常，`&<<` 静默 |
| bigint 移位支持 | ✅ | N/A | N/A |
| 无符号右移 `>>>` | ✅ 原生 | ✅ 原生 | ❌ 需自定义 |
| 浮点操作数截断 | double→long, float→int | double→long, float→int | ❌ 无此规则 |
| long 字面量后缀 | ❌ 不支持 `L` | ✅ `L` 后缀 | ✅ 类型推断 |

## 4. 用例 1:1 对照

### ArkTS 用例 vs Java/Swift 等价代码

**用例 1: int 左移**

|------|------|
| ArkTS | `let r: int = 5 << 2; // 20` |

| Swift | `var r = 5 &<< 2 // 20` |

**用例 2: int 符号右移**
| 语言 | 代码 |
|------|------|
| ArkTS | `let r: int = -20 >> 2; // -5` |
| Java | `int r = -20 >> 2; // -5` |
| Swift | `var r = -20 >> 2 // -5` |

**用例 3: int 无符号右移**
| 语言 | 代码 |
|------|------|
| ArkTS | `let r: int = -1 >>> 31; // 1` |
| Java | `int r = -1 >>> 31; // 1` |
| Swift | `func urs(n: Int32, s: Int32) -> Int32 { UInt32(bitPattern: n) >> UInt32(s & 0x1f) }` |

**用例 4: 移位距离掩码**

**用例 5: bigint 移位**

|------|------|
| ArkTS | `let r: bigint = 5n << 2n; // 20n` |

### 关键差异详解

#### 1. Swift 无 `>>>` 运算符 ⭐

| 语言 | 代码 | 行为 |
|------|------|------|
| ArkTS | `-1 >>> 31` | `1`（原生支持） |
| Java | `-1 >>> 31` | `1`（原生支持） |
| Swift | 无对应运算符 | 需自定义：`Int32(bitPattern: UInt32(bitPattern: n) >> UInt32(s & 0x1f))` |

#### 2. Swift 默认溢出检测 ⭐

| 语言 | `Int32.max << 1` | 行为 |
|------|-----------------|------|
| ArkTS | `2147483647 << 1` | `-2`（静默截断） |
| Java | `Integer.MAX_VALUE << 1` | `-2`（静默截断） |
| Swift | `Int32.max << 1` | ❌ 运行时错误（溢出检测） |
| Swift | `Int32.max &<< 1` | `-2`（`&<<` 允许溢出） |

#### 3. ArkTS 独有特性：bigint 移位 ⭐

| 语言 | `1n << 100n` | 行为 |
|------|-------------|------|
| ArkTS | `1n << 100n` | `1267650600228229401496703205376n` ✅ |
| Java | N/A | 无 bigint 原始类型移位 |
| Swift | N/A | 无 bigint 原始类型 |

#### 4. ArkTS 不支持 `L` 后缀 ⭐

| 语言 | long 字面量 | 示例 |
|------|------------|------|
| ArkTS | ❌ 不支持 `L` | `let v: long = 8;`（需变量声明） |
| Java | ✅ | `long v = 8L;` |
| Swift | ✅ | `let v = 8 as Int64` |

| 维度 | ArkTS | Java | Swift |
|------|:-----:|:----:|:-----:|

| 数值类型覆盖 | ⭐⭐⭐⭐⭐（含 bigint） | ⭐⭐⭐⭐（无 bigint） | ⭐⭐⭐（无 >>>） |

## 7. 核心结论

1. **ArkTS ≈ Java 完全一致**：移位运算符行为、距离掩码、溢出处理与 Java 完全相同
2. **Swift 差异**：
   - 默认溢出检测（需 `&<<` 而非 `<<`）
   - 无 `>>>` 运算符
   - Int 默认 64 位（Int 与 Int32 行为不同）
3. **ArkTS 独有能力**：bigint 移位运算（Java/Swift 无对应）
4. **ArkTS 限制**：不支持 `L` 后缀，需用变量声明创建 long 类型

## 8. ArkTS 设计建议

1. **保持现状**：ArkTS 移位实现与 Java 一致，符合 Java 开发者预期
2. **ArkTS spec 中 `>>>` 公式**：文档行文中的 `(2L << ~s)` 可考虑删除 `L` 后缀以符合 ArkTS 语法
3. **bigint >>> 的编译时检查**：正确实现（bigint >>> → compile-time error），建议保持
