# 7.35.1 Lambda Signature — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: Lambda 语法形式差异 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_35_01_001~007（全部 PASS 用例） |
| **实测结果** | 三语言 Lambda 语法形式不同 |
| **差异类型** | 设计差异（纯语法风格差异，语义等价） |

**描述**：Lambda 表达式语法形式在三语言中不同：ArkTS 使用 `=>` 箭头语法，Java 使用 `->` 箭头语法，Swift 使用块语法加 `in` 关键字分隔签名和体。

**跨语言对比**：

| 语言 | 语法 | 示例 |
|------|------|------|
| ArkTS | `(params) => body` 或 `param => body` | `(x: int) => x * 2` |
| Java | `(params) -> body` 或 `param -> body` | `(Integer x) -> x * 2` |
| Swift | `{ (params) -> T in body }` | `{ (x: Int) -> Int in x * 2 }` |

**分类**：跨语言设计差异

**建议**：当前设计合理，无变更需要。

### ID-02: 显式返回类型注解支持差异 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_35_01_005_PASS_RETURN_TYPE_ANNOTATION |
| **实测结果** | ArkTS 和 Swift 支持显式返回类型注解，Java 不支持 |
| **差异类型** | 设计差异（Java 无对应能力） |

**描述**：Lambda 签名中是否支持显式指定返回类型。ArkTS 使用 `(x: int): int => x * 2` 语法，Swift 使用 `{ (x: Int) -> Int in x * 2 }`，Java 无显式返回类型语法（返回类型由目标函数式接口决定）。

**跨语言对比**：

| 语言 | 代码 | 行为 |
|------|------|------|
| ArkTS | `(x: int): int => x * x` | ✅ 支持显式返回类型注解 |
| Java | `(Integer x) -> x * x` | ❌ 无显式返回类型语法，由函数式接口推断 |
| Swift | `{ (x: Int) -> Int in x * x }` | ✅ 支持显式返回类型注解 |

**分类**：跨语言设计差异

**建议**：ArkTS 的返回类型注解语法与 Swift 类似，比 Java 更完善，建议保留当前设计。

### ID-03: 简写参数名支持差异 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | 无（ArkTS 无此特性） |
| **实测结果** | Swift 支持，ArkTS 和 Java 不支持 |
| **差异类型** | 设计差异（Swift 独有语法糖，非必要功能） |

**描述**：Lambda 是否支持基于位置的简写参数名。Swift 原生支持 `$0, $1, $2...` 基于位置的简写参数名，ArkTS 和 Java 不支持简写参数名，必须给参数命名。

**跨语言对比**：

| 语言 | 代码 | 行为 |
|------|------|------|
| ArkTS | 不支持简写参数名 | 必须给参数命名 |
| Java | 不支持简写参数名 | 必须给参数命名 |
| Swift | `{ $0 + $1 }` | 原生支持 `$0, $1, $2...` 基于位置的简写参数名 |

**分类**：跨语言设计差异

**建议**：当前设计合理，无需引入 `$0` 简写语法。

### D 类详情

当前无 D 类问题。Spec 7.35.1 的所有规则（形式参数签名、类型推断、重名参数错误、无上下文错误）均与编译器实现一致。全部 12 个用例通过。

- **重复参数名错误**：`(x: int, x: int) => x` → ✅ compile-fail（EXP_07_35_01_008）
- **无法推断类型错误**：`const f = x => x` → ✅ compile-fail（EXP_07_35_01_010）
- **类型推断成功场景**：从目标类型和泛型上下文均能正确推断（EXP_07_35_01_001/006/007）
