# 7.6.4 Object Literal Evaluation — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: 对象字面量求值语法（ArkTS 独有）

| 字段 | 值 |
|------|-----|
| **差异类型** | 语言特性差异 |
| **合理性** | 符合各自语言设计 |

**描述**：ArkTS 是唯一支持专用对象字面量求值语法的语言。Java/Swift 使用构造器 + 字段赋值模拟，但逻辑等价。

**跨语言对比**：

| 语言 | 对象字面量语法 |
|------|--------------|
| ArkTS | ✅ 原生支持 `{f: expr}` |
| Java | ❌ 需 `new` + 赋值 |
| Swift | ❌ 需 `init()` + 赋值 |

**分类**：符合 ArkTS spec 的语言设计差异

---

### ID-02: 构造器先于字段赋值的语义一致性

| 字段 | 值 |
|------|-----|
| **差异类型** | 语义等价 |
| **合理性** | 三种语言均遵循相同语义 |

**描述**：ArkTS（构造器→字段赋值）、Java（`new` → `p.field = val`）、Swift（`init()` → `obj.prop = val`）均遵循先调用构造器再执行字段赋值的语义，最终字段值以赋值为准。

**跨语言对比**：

| 语言 | 语义顺序 |
|------|---------|
| ArkTS | 构造器 → 字段赋值 |
| Java | `new` → 字段赋值 |
| Swift | `init()` → 属性赋值 |

**分类**：符合 ArkTS spec 的语言设计差异

---

### ID-03: 左到右求值顺序

| 字段 | 值 |
|------|-----|
| **差异类型** | 语义等价 |
| **合理性** | 三种语言均按程序顺序执行 |

**描述**：ArkTS 对象字面量字段从左到右求值。Java/Swift 按语句顺序执行字段赋值，同样遵循程序顺序语义。

**跨语言对比**：

| 语言 | 求值顺序 |
|------|---------|
| ArkTS | 对象字面量字段从左到右 |
| Java | 语句执行顺序 |
| Swift | 语句执行顺序 |

**分类**：符合 ArkTS spec 的语言设计差异

---

### ID-04: 异常完成传播

| 字段 | 值 |
|------|-----|
| **差异类型** | 语义等价 |
| **合理性** | 三种语言均支持异常传播 |

**描述**：构造器或字段表达式抛出异常时，三语言均终止字面量/赋值求值过程，异常向外传播。

**跨语言对比**：

| 语言 | 构造器异常 | 字段异常 |
|------|-----------|---------|
| ArkTS | 字面量终止 | 后续字段跳过 |
| Java | `new` 终止 | 后续赋值跳过 |
| Swift | `init()` 终止 | 后续赋值跳过 |

**分类**：符合 ArkTS spec 的语言设计差异
