# 7.6.4 Object Literal Evaluation — 三语言对比报告

## 1. 概览

Object Literal Evaluation 定义了对象字面量的运行时求值语义。ArkTS 采用"构造器先执行→字段从左到右求值赋值"的标准化流程：

| 语言 | 定位 |
|------|------|
| **ArkTS** | 原生对象字面量求值语义：构造器(无参)→字段表达式左到右求值→赋值。支持异常完成传播 |
| **Java** | 无对象字面量语法，通过 `new` 构造器 + 字段语句模拟等价逻辑 |
| **Swift** | 无对象字面量语法，通过 `init()` 构造器 + 属性赋值模拟等价逻辑 |

## 2. 章节对应关系

| 语义步骤 | ArkTS | Java | Swift |
|---------|-------|------|-------|
| 创建实例 | `let x: C = {...}` | `C x = new C()` | `let x = C()` |
| 构造器先执行 | 无参构造器先被调用 | 构造器在 `new` 时执行 | `init()` 先被调用 |
| 字段顺序求值 | 左到右文本顺序 | 语句执行顺序 | 语句执行顺序 |
| 字段赋值 | 每个字段表达式→对应字段 | `x.field = expr` | `obj.prop = expr` |
| 构造器异常 | 字面量求值终止 | `new` 表达式异常抛出 | `init()` 异常抛出 |
| 字段表达式异常 | 后续字段不执行 | 后续语句不执行 | 后续语句不执行 |
| 构造器值被覆盖 | 字段赋值覆盖构造器值 | 赋值覆盖构造器值 | 赋值覆盖构造器值 |

## 3. 关键差异矩阵

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 对象字面量求值语法 | ✅ 原生支持 `{f: expr}` | ❌ 需 `new` + 赋值 | ❌ 需 `init()` + 赋值 |
| 构造器→字段顺序 | ✅ 严格保证 | ✅ 严格保证 | ✅ 严格保证 |
| 左到右求值顺序 | ✅ 严格文本顺序 | ✅ 语句执行顺序 | ✅ 语句执行顺序 |
| 异常传播 (构造器) | ✅ 字面量终止 | ✅ `new` 表达式终止 | ✅ `init()` 终止 |
| 异常传播 (字段) | ✅ 后续字段跳过 | ✅ 后续赋值跳过 | ✅ 后续赋值跳过 |
| 构造器默认值覆盖 | ✅ 字段值优先生效 | ✅ 赋值值优先生效 | ✅ 赋值值优先生效 |

## 4. 用例对照

### 4.1 对象字面量求值（ArkTS 独有语法）

| 语言 | 代码 |
|------|------|
| ArkTS | `let x: Data = {a: funcA(), b: funcB()}` — 无参构造器 → funcA() → 赋值 → funcB() → 赋值 |
| Java | `Data x = new Data(); String av = funcA(); x.a = av; String bv = funcB(); x.b = bv;` |
| Swift | `let x = Data(); let av = funcA(); x.a = av; let bv = funcB(); x.b = bv` |

### 4.2 构造器异常传播

| 语言 | 构造器抛出异常 |
|------|--------------|
| ArkTS | 对象字面量求值立即终止，后续字段不执行 |
| Java | `new` 表达式立即抛出，后续赋值不执行 |
| Swift | `try C()` 立即抛出，后续属性赋值不执行 |

### 4.3 构造器值被字段覆盖

| 语言 | 构造器设 value=42，字段赋 value=100 |
|------|------------------------------------|
| ArkTS | `{value: 100}` → value == 100 ✅ |
| Java | `obj.value = 100` → value == 100 ✅ |
| Swift | `obj.value = 100` → value == 100 ✅ |

### 4.4 左到右求值顺序

| 语言 | 顺序追踪结果 |
|------|-------------|
| ArkTS | `{a: setA(), b: setB(), c: setC()}` → order "ABC" ✅ |
| Java | `av = setA(); bv = setB(); cv = setC()` → order "ABC" ✅ |
| Swift | `av = setA(); bv = setB(); cv = setC()` → order "ABC" ✅ |

## 7. 核心结论

1. **三语言求值语义完全等价**：ArkTS 构造器先于字段执行、字段从左到右求值、异常传播等语义，在 Java 和 Swift 中有完全对等的语义行为。
2. **ArkTS 的语法优势**：语法层面封装了构造器调用 + 字段赋值流程，使代码更简洁。
3. **异常传播语义一致**：三个语言均支持构造器异常和字段表达式异常的正确传播，后续字段/赋值被跳过。
4. **构造器默认值覆盖行为一致**：三语言中，字段赋值优先级均高于构造器设置的默认值。

## 8. ArkTS 设计建议

- 当前设计完善，无缺陷。
- 求值语义（构造器先 → 字段左到右 → 异常终止）是标准的 OOP 求值模型。
- 建议未来考虑文档化求值顺序的保证，明确字段表达式可依赖之前字段的赋值结果。
