# 7.14 Chaining Operator — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: 链式操作符语法支持 — ArkTS > Java ⭐

| 字段 | 值 |
|------|-----|
| **问题描述** | ArkTS 和 Swift 原生支持 `?.` 链式操作符；Java 无对应语法 |
| **分类** | 语言设计差异 |

**描述**：链式操作符 `?.` 用于安全访问可能为 nullish 的值。ArkTS 支持四种上下文：字段访问（`obj?.field`）、方法调用（`obj?.method()`）、索引访问（`arr?.[i]`）和函数调用（`fn?.()`）。Swift 支持除函数调用外相同的语法（Swift 函数调用为 `fn?()`）。Java 无此运算符，需手动 null 检查。

**跨语言对比**：

| 语言 | 语法 | 缺失返回值 | 结果类型 |
|------|------|-----------|---------|
| ArkTS | `obj?.field`、`arr?.[i]`、`fn?.()` | `undefined` | `T \| undefined` |
| Java | ❌ 无运算符（手写 if-null） | `null` | T 或 null |
| Swift | `obj?.field`、`arr?.[i]`、`fn?()` | `nil` | `T?` (Optional) |

**结论**：符合各自语言的 spec 定义，ArkTS 与 Swift 语法高度一致。

### ID-02: 静态方法限制 — ArkTS 独有 ⭐⭐

| 字段 | 值 |
|------|-----|
| **问题描述** | ArkTS 明确禁止对静态方法使用 `?.`（编译时错误）；Java 和 Swift 无此限制 |
| **分类** | 符合 ArkTS spec 的独特设计 |

**描述**：ArkTS 编译器对 `A?.f()`（静态方法）报编译时错误。这是因为静态方法调用不需要实例，不存在 null 安全问题，`?.` 在此场景中无意义。Java 和 Swift 不对静态方法使用 `?.` 作限制。

**跨语言对比**：

| 语言 | 代码 | 结果 |
|------|------|------|
| ArkTS | `A?.f()` (静态) | ❌ 编译时错误 |
| Java | `Class.staticMethod()` | ✅ 编译通过（无 null 问题）|
| Swift | `Type.staticMethod()` | ✅ 编译通过（无 nil 问题）|

**原因**：静态方法调用不需要实例，不存在 null 安全问题，`?.` 不适用。

### ID-03: LHS 和递增/递减限制 — 三语言一致 ⭐

| 字段 | 值 |
|------|-----|
| **问题描述** | 三语言都禁止链式操作符出现在赋值左侧或递增/递减表达式中 |
| **分类** | 三语言一致 |

**描述**：ArkTS、Java 和 Swift 均不允许 `x?.y = value`、`x?.y++`、`++x?.y`、`x?.y--` 和 `--x?.y` 等表达式通过编译。

**跨语言对比**：

| 场景 | ArkTS | Java | Swift |
|------|-------|------|-------|
| `x?.y = v` | ❌ 编译错误 | ❌ 无运算符 | ❌ 编译错误 |
| `x?.y++` | ❌ 编译错误 | ❌ 无运算符 | ❌ 编译错误 |

**结论**：符合各语言 spec。

### ID-04: 非 nullish 类型无效果 — ArkTS 和 Swift 一致

| 字段 | 值 |
|------|-----|
| **问题描述** | 非 nullish 类型上使用 `?.` 不影响结果类型（不会变成 Optional） |
| **分类** | ArkTS ≈ Swift 一致 |

**描述**：当操作符 `?.` 应用于已知非 nullish 类型时，ArkTS 和 Swift 均不改变结果类型，不会使结果变为可选类型。

**跨语言对比**：

| 语言 | 代码 | 结果类型 |
|------|------|---------|
| ArkTS | `s?.[0]` (s: string) | `string`（非 `string\|undefined`）|
| Swift | `s?[0]` (s: String) | `Character`（非 `Character?`）|

**结论**：符合各自语言的 spec。

---

### ID-05: 编译时警告 — ArkTS 独有 ⭐

| 字段 | 值 |
|------|-----|
| **问题描述** | ArkTS 在表达式已知始终 nullish 或始终非 nullish 时发出编译时警告 |
| **分类** | ArkTS 独有机制 |

**描述**：当 `?.` 左侧表达式被静态类型推断为始终非 nullish 或始终 nullish 时，ArkTS 编译器发出警告：
- 始终非 nullish：`c?.f` → warning: 总为非 nullish
- 始终 nullish：`d?.f` → warning: 总为 undefined

Java 和 Swift 无此编译时警告机制。
