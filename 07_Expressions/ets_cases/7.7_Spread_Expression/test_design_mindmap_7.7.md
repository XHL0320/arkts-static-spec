# 7.7 Spread Expression - 测试设计思维导图

## 概述

扩展表达式（Spread Expression）定义了 `...` 扩展运算符在数组字面量和函数调用中的语义：数组/元组/字符串扩展、函数调用扩展、复制语义、只读源数组、编译时错误检测。

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 基本 int[] 数组扩展 | int[] 数组基本扩展编译通过 |
| 002 | FixedArray 扩展 | FixedArray&lt;int&gt; 扩展编译通过 |
| 003 | 多扩展合并 | `[...a1, ...a2]` 多扩展编译通过 |
| 004 | 扩展+字面量混合 | 扩展+字面量+扩展混合编译通过 |
| 005 | 函数调用扩展 | `foo(...arr)` 基本函数调用编译通过 |
| 006 | 多扩展函数调用 | `foo(...a1, ...a2)` 多扩展调用编译通过 |
| 007 | 函数返回值扩展 | `foo(...g1())` 函数返回值扩展编译通过 |
| 008 | 嵌套扩展 | `[...[...arr]]` 嵌套扩展编译通过 |
| 009 | 只读源数组扩展 | 只读源数组扩展编译通过 |
| 010 | 元组扩展函数调用 | `bar(...tuple)` 元组扩展调用编译通过 |
| 011 | 字符串类型扩展 | 字符串类型扩展编译通过 |
| 012 | 元组扩展字面量 | 元组扩展在字面量中编译通过 |

### compile-fail

| # | 测试点 | 说明 |
|---|--------|------|
| 013 | 非 iterable 类扩展 | 非 iterable 类扩展 → 编译时错误 |
| 014 | 非 iterable 数值扩展 | int 数值扩展 → 编译时错误 |
| 015 | 数组扩展到非 rest 参数 | 数组扩展到非 rest 参数 → 编译时错误 |
| 016 | 元组扩展到非 rest 参数 | 元组扩展到非 rest 参数 → 编译时错误 |

### runtime（验证实际运行时行为）

| # | 测试点 | 预期值 |
|---|--------|--------|
| 017 | 基本扩展值正确 | 扩展后数组元素正确 |
| 018 | 复制语义（互不影响） | 修改新数组不影响原数组 |
| 019 | 多扩展合并值正确 | 多扩展合并后数组元素正确 |
| 020 | 扩展混合字面量 | 扩展混合字面量元素正确 |
| 021 | 函数调用扩展 | 函数调用扩展正确传递参数 |
| 022 | 只读源扩展 | 只读源扩展后目标数组可读写 |

## 文件命名规范

`EXP_07_07_YYY_{CATEGORY}_{DESCRIPTION}.ets`

- YYY: 001 起连续编号
- CATEGORY: PASS / FAIL / RUNTIME
- DESCRIPTION: 大写+下划线描述

## 注释模板要求

每个文件必须包含 5 个 JSDoc tag：@id, @expect, @section, @design, @note
