# 7.7 Spread Expression — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: 扩展表达式语法（ArkTS 独有）

| 字段 | 值 |
|------|-----|
| **差异类型** | 语言特性差异 |
| **合理性** | 符合各自语言设计 |

**描述**：ArkTS 是唯一支持 `...` 扩展表达式语法的语言。Java 使用 `Collections.addAll` / `Stream.concat`，Swift 使用 `+` 运算符合并数组。

**跨语言对比**：

| 语言 | 扩展语法 |
|------|---------|
| ArkTS | ✅ `...` 扩展语法 |
| Java | ❌ 需手动合并 |
| Swift | ❌ 使用 `+` 运算符 |

**分类**：符合 ArkTS spec 的语言设计差异

### ID-02: Rest 参数 vs Varadic 参数语义等价

| 字段 | 值 |
|------|-----|
| **差异类型** | 语法机制差异 |
| **合理性** | 三种语言均支持 rest/variadic 参数 |

**描述**：ArkTS rest 参数（`...args: int[]`）、Java varargs（`Integer... args`）、Swift variadic（`_ p: Int...`）语义等价。但 Java 和 Swift 不支持将数组扩展为 individual arguments 传递给 rest 参数的语法。

**跨语言对比**：

| 语言 | Rest 参数声明 | 调用语法 |
|------|--------------|---------|
| ArkTS | `...args: int[]` | `func(...arr)` |
| Java | `Integer... args` | `func(arr)` |
| Swift | `_ p: Int...` | `func(1, 2, 3)` |

**分类**：符合 ArkTS spec 的语言设计差异

### ID-03: 扩展在字面量中使用（ArkTS 独有）

| 字段 | 值 |
|------|-----|
| **差异类型** | 语言特性差异 |
| **合理性** | ArkTS 数组字面量支持扩展语法 |

**描述**：ArkTS 允许 `[...arr1, ...arr2]` 在数组字面量中扩展数组。Java 需 `List` 合并操作，Swift 需 `arr1 + arr2`。

**跨语言对比**：

| 语言 | 字面量扩展 |
|------|-----------|
| ArkTS | ✅ `[...a1, ...a2]` |
| Java | ❌ 需合并操作 |
| Swift | ❌ 需 `+` 运算 |

**分类**：符合 ArkTS spec 的语言设计差异

### ID-04: 复制语义一致

| 字段 | 值 |
|------|-----|
| **差异类型** | 语义等价 |
| **合理性** | 三种语言均支持值复制语义 |

**描述**：ArkTS、Java、Swift 中扩展/数组合并均执行值复制，修改新数组不影响原数组。

**跨语言对比**：

| 语言 | 复制行为 |
|------|---------|
| ArkTS | `let b = [...a]; b[0]=99` → a[0] 不变 |
| Java | `Integer[] b = a.clone(); b[0]=99` → a[0] 不变 |
| Swift | `var b = a; b[0]=99` → a[0] 不变 |

**分类**：符合 ArkTS spec 的语言设计差异

**分类**：符合 ArkTS spec 的语言设计差异
