### ID-01: 索引类型检查 — 三语言完全一致

| 字段 | 值 |
|------|-----|
| **问题描述** | 三语言对数组索引的类型检查行为完全一致：仅整数类型（int/Int）可作为索引 |
| **分类** | 三语言一致，无设计差异 |
| **结论** | 符合各自语言的 spec 定义 |

| 场景 | ArkTS | Java | Swift |

| long 无转换索引 | ❌ 编译错误 | ❌ 编译错误 | ❌ 编译错误 |
| float 无转换索引 | ❌ 编译错误 | ❌ 编译错误 | ❌ 编译错误 |

### ID-02: 越界运行时异常差异 ⭐

| 字段 | 值 |
|------|-----|
| **问题描述** | 三语言数组越界均抛出运行时异常，但异常类型不同 |
| **分类** | 符合 ArkTS spec 的语言设计差异 |
| **严重性** | ⭐（低）—— 运行时可捕获性不同 |

**跨语言对比**：

| 语言 | 越界行为 | 可捕获 |
|------|---------|--------|
| ArkTS | `RangeError` | ✅ `try/catch` |
| Java | `ArrayIndexOutOfBoundsException` | ✅ `try/catch` |
| Swift | `fatal error` | ❌ 不可捕获 |
