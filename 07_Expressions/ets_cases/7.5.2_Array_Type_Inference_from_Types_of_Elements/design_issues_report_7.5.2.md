# 7.5.2 Array Type Inference from Types of Elements — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: 数值类型混合推断策略 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_05_02_003_PASS_NUMERIC_TYPES_MIXED |
| **实测结果** | ArkTS 推断为 `number[]`，Java 推断为 `double[]`，Swift 歧义需标注 |
| **错误信息** | 无（类型推断策略差异） |

**描述**：三语言处理数值混合推断的策略不同。

**跨语言对比**：

| 语言 | `[1, 2.1, 3]` 推断结果 | 说明 |
|------|------------------------|------|
| ArkTS | `number[]` | 统一用 number 类型 |
| Java | `double[]` | int 自动提升为 double |
| Swift | 歧义，需 `as [Double]` | 类型不统一，需显式标注 |

**分类**：语言设计差异（符合各自语言设计）

**建议**：ArkTS 的 `number[]` 推断简洁合理，无需改动。

### ID-02: 空数组处理 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_05_02_011_FAIL_EMPTY_ARRAY |
| **实测结果** | ArkTS 和 Swift 禁止无上下文空数组，Java 允许 |
| **错误信息** | ArkTS 编译时错误：类型无法推断 |

**描述**：ArkTS 和 Swift 禁止无上下文的空数组字面量，Java 允许。

**跨语言对比**：

| 语言 | `[]` 无上下文 | 原因 |
|------|-------------|------|
| ArkTS | ❌ 编译时错误 | 类型无法推断 |
| Java | ✅ `new int[0]` 或 `{}`（有声明类型） | 类型从声明/初始化器推断 |
| Swift | ❌ 编译时错误 | 类型无法推断 |

**分类**：编译时安全检查策略差异（符合各自语言设计）

**建议**：保持当前设计，空数组编译时错误提高了类型安全性。

### ID-03: 字面量类型提升 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_05_02_005_PASS_LITERAL_TYPE_PROMOTION |
| **实测结果** | ArkTS 将字面量 union 自动提升为超类型 string |
| **错误信息** | 无（语言特性差异） |

**描述**：ArkTS 支持字符串字面量类型（如 `"A" | "B"`），在数组推断中自动提升为 `string`。Java 和 Swift 不支持字面量类型作为类型系统中的一等公民。

**跨语言对比**：

| 语言 | 字面量类型支持 | 数组推断行为 |
|------|---------------|-------------|
| ArkTS | ✅ 支持字符串字面量类型 | 自动提升为 string |
| Java | ❌ 不支持 | N/A |
| Swift | ❌ 不支持 | N/A |

**分类**：语言特性差异（符合 ArkTS 规范）

**建议**：字面量类型提升是 ArkTS 的独特优势，保留当前设计。

### ID-04: 混合类型的表达方式 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_05_02_004_PASS_MIXED_STRING_NUMBER |
| **实测结果** | ArkTS 用 `(string\|number)[]`，Java 降级为 `Object[]`，Swift 用 `[Any]` |
| **错误信息** | 无（类型系统表达力差异） |

**描述**：ArkTS 使用 union 类型（`(string | number)[]`）保留每个元素的精度信息。Java 统一降级为 `Object[]` 丢失类型信息。Swift 使用 `[Any]` 同样丢失类型信息。

**跨语言对比**：

| 语言 | 混合类型表示 | 类型精度 |
|------|-------------|---------|
| ArkTS | `(string\|number)[]` | 高（保留元素类型信息） |
| Java | `Object[]` | 低（丢失类型信息） |
| Swift | `[Any]` | 低（丢失类型信息） |

**分类**：类型系统表达力差异（符合各自语言设计）

**建议**：union 数组类型保留精度信息，优于 Java/Swift 的设计，值得保留。
