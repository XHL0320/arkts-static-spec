# 7.8 Parenthesized Expression — 三语言对比报告

## 1. 概览

Parenthesized Expression 是三种语言中语义完全一致的表达式：

| 语言 | 语法 | 语义 |
|------|------|------|
| **ArkTS** | `(expr)` | 类型和值与 expr 相同，仅改变求值顺序 |
| **Java** | `(expr)` | 类型和值与 expr 相同，仅改变求值顺序 |
| **Swift** | `(expr)` | 类型和值与 expr 相同，仅改变求值顺序 |

## 2. 章节对应关系

| 规则 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 语法 | `(expr)` | `(expr)` | `(expr)` |
| 类型 | 与 expr 相同 | 与 expr 相同 | 与 expr 相同 |
| 值 | 与 expr 相同 | 与 expr 相同 | 与 expr 相同 |
| 分组 | `(1+2)*3` → 9 | `(1+2)*3` → 9 | `(1+2)*3` → 9 |
| 嵌套 | `(((1+2)))` → 3 | `(((1+2)))` → 3 | `(((1+2)))` → 3 |
| 空括号 | `()` ❌ 编译时错误 | `()` ❌ 编译时错误 | `()` ❌ 编译时错误 |
| 函数参数 | `foo((expr))` | `foo((expr))` | `foo((expr))` |
| 条件 | `if ((x>0))` | `if ((x>0))` | `if ((x>0))` |
| 数组元素 | `[(1+2)]` | `{(1+2)}` | `[(1+2)]` |

## 3. 关键差异矩阵

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 括号分组 | ✅ | ✅ | ✅ |
| 值不变性 | ✅ | ✅ | ✅ |
| 类型不变性 | ✅ | ✅ | ✅ |
| 嵌套括号 | ✅ | ✅ | ✅ |
| 空括号错误 | ✅ 编译时错误 | ✅ 编译时错误 | ✅ 编译时错误 |
| 条件中括号 | ✅ | ✅ | ✅ |

**差异：0** — 三种语言语义完全一致。

## 4. 用例对照

### 4.1 基本算术括号分组

| 语言 | 代码 | 结果 |
|------|------|------|
| ArkTS | `(1 + 2) * 3` | `9` |
| Java | `(1 + 2) * 3` | `9` |
| Swift | `(1 + 2) * 3` | `9` |

三语言括号表达式语义完全等价。

### 4.2 值不变性

| 语言 | 代码 | 说明 |
|------|------|------|
| ArkTS | `let x = (42)` | x = 42，括号仅分组 |
| Java | `int x = (42)` | x = 42 |
| Swift | `let x = (42)` | x = 42 |

括号表达式不改变表达式的值和类型。

### 4.3 空括号编译时错误

| 语言 | 代码 | 行为 |
|------|------|------|
| ArkTS | `()` | ❌ 编译时错误 |
| Java | `()` | ❌ 编译时错误 |
| Swift | `()` | ❌ 编译时错误 |

三语言均禁止空括号表达式。

## 7. 核心结论

1. **三语言括号表达式语义完全等价**：类型、值、分组行为完全一致。
2. **空括号均报编译时错误**：`()` 不是有效表达式。
3. **括号不影响性能或语义**：仅作为语法分组工具。

## 8. ArkTS 设计建议

- 括号表达式语义与 Java/Swift 完全一致，是成熟的语法设计。
- 无需调整现有行为。
