# 7.8 Parenthesized Expression — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: 括号表达式语义完全一致

| 字段 | 值 |
|------|-----|
| **差异类型** | 语义等价 |
| **合理性** | 三种语言均相同 |

**描述**：ArkTS、Java、Swift 的括号表达式语义完全一致：`(expr)` 不改变表达式的类型和值，仅改变求值顺序。

**跨语言对比**：

| 语言 | 括号语义 |
|------|---------|
| ArkTS | `(expr)` 不改变类型和值 |
| Java | `(expr)` 不改变类型和值 |
| Swift | `(expr)` 不改变类型和值 |

**分类**：符合 ArkTS spec 的语言设计差异

### ID-02: 空括号处理

| 字段 | 值 |
|------|-----|
| **差异类型** | 语义等价 |
| **合理性** | 均报编译时错误 |

**描述**：三种语言中空括号 `()` 均不是有效的值表达式，编译时报错。

**跨语言对比**：

| 语言 | 空括号行为 |
|------|-----------|
| ArkTS | ❌ 编译时错误 |
| Java | ❌ 编译时错误 |
| Swift | ❌ 编译时错误 |

**分类**：符合 ArkTS spec 的语言设计差异
