# 7.8 Parenthesized Expression - 测试设计思维导图

## 概述

括号表达式语义：`(expr)` 不改变类型和值，仅改变求值顺序。

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 基本算术分组 | `(1+2)*3` 括号改变优先级编译通过 |
| 002 | 多层嵌套 | `(((1+2)))` 多层嵌套括号编译通过 |
| 003 | 函数调用括号 | `foo((a+b))` 函数参数中括号编译通过 |
| 004 | 条件语句括号 | `if ((x>0))` 条件语句中括号编译通过 |
| 005 | 数组元素括号 | `[(1+2)]` 数组字面量中括号元素编译通过 |
| 006 | 字符串/布尔值括号 | `("hello")`, `(true)` 括号表达式编译通过 |

### compile-fail

| # | 测试点 | 说明 |
|---|--------|------|
| 007 | 空括号 | `()` 不是有效表达式 → 编译时错误 |

### runtime（验证实际运行时行为）

| # | 测试点 | 预期值 |
|---|--------|--------|
| 008 | 值不变性 | `(expr) == expr` 括号不改变值 |
| 009 | 分组正确 | `(1+2)*3 == 9` 分组改变求值顺序 |
| 010 | 多类型括号等价性 | 多类型括号表达式等价验证 |

## 文件命名规范

`EXP_07_08_YYY_{CATEGORY}_{DESCRIPTION}.ets`

- YYY: 001 起连续编号
- CATEGORY: PASS / FAIL / RUNTIME
- DESCRIPTION: 大写+下划线描述

## 注释模板要求

每个文件必须包含 5 个 JSDoc tag：@id, @expect, @section, @design, @note
