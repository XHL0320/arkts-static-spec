# 7.9 this Expression — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: this/self 关键字名称不同

| 字段 | 值 |
|------|-----|
| **差异类型** | 语法差异 |
| **合理性** | 语言传统差异 |

**描述**：ArkTS/Java 使用 `this`，Swift 使用 `self`。语义完全一致。

**跨语言对比**：

| 语言 | 关键字 |
|------|--------|
| ArkTS | `this` |
| Java | `this` |
| Swift | `self` |

**分类**：符合 ArkTS spec 的语言设计差异

---

### ID-02: 对象字面量 lambda 的 this

| 字段 | 值 |
|------|-----|
| **差异类型** | 语言特性差异 |
| **合理性** | 符合各自语言设计 |

**描述**：ArkTS 规定对象字面量内 lambda 的 `this` 指向包围类而非字面量。Java 中 lambda 的 `this` 也指向包围类（语义一致）。Swift 闭包需显式捕获 `self`。

**跨语言对比**：

| 语言 | Lambda/闭包 this 指向 |
|------|---------------------|
| ArkTS | 指向包围类 |
| Java | 指向包围类 |
| Swift | 需显式捕获 `self` |

**分类**：符合 ArkTS spec 的语言设计差异
