## 设计问题及差异清单

### ID-01: Swift `|` 运算符对 Bool 类型不可用

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_31_006_RUNTIME_COND_OR_SEMANTICS |
| **实测结果** | Swift 中 `|` 对 `Bool` 类型不可用，无法比较 `||` 与 `|` 的一致性 |
| **差异类型** | 跨语言设计差异 |

**描述**：Swift 中 `|` 仅用于整数按位或，不适用于布尔类型。`Bool` 类型仅支持 `||`（短路 OR）、`&&`（短路 AND）、`!`（NOT）。这与 ArkTS/Java 不同，后者的 `|` 对 boolean 也有效。

**跨语言对比**：

| 语言 | 代码 | 行为 |
|------|------|------|
| ArkTS | `true || true == true | true` | `||` 与 `|` 对 boolean 可比较一致性 |
| Java | `true || true == true | true` | `||` 与 `|` 对 boolean 可比较一致性 |
| Swift | `\|` 对 `Bool` 类型不可用 | 仅 `Int` 支持位运算，无法比较 `\|\|` 与 `\|` 的一致性 |
