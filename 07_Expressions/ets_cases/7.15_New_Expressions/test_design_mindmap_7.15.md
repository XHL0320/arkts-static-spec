# 7.15 New Expressions - 测试设计思维导图

## 概述

类实例创建表达式（new 表达式）用于创建类的新实例并调用构造器初始化实例。本节验证 ArkTS 中 `new` 表达式的编译通过、编译时错误和运行时行为。

new 表达式执行流程：
1. 创建类的新实例
2. 调用构造器初始化实例
3. 返回新实例（或异常完成）

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 默认构造器 `new A()` | 无参无构造器，编译通过 |
| 002 | 带参数构造器 `new A(42)` | int 参数构造器，编译通过 |
| 003 | 创建并存储变量 | new 结果赋值给变量 |
| 004 | 方法链 `new A().method()` | new 后直接调用方法（有括号） |
| 005 | 括号包裹 `(new A).method()` | 括号包裹 new 后调用方法 |
| 006 | 无括号 `new A` | 无括号无参数创建实例 |
| 007 | 泛型实例化 `new MyBox<int>(42)` | 泛型类 + 类型参数 |

### compile-fail

| # | 测试点 | 说明 |
|---|--------|------|
| 008 | 缺少括号 `new A.method()` | 缺少括号导致语法错误 |
| 009 | 类型参数 `new T()` | 类型参数作为 typeReference 编译时错误 |
| 010 | FixedArray<T> 类型参数 | FixedArray 元素类型为类型参数编译时错误 |
| 011 | 接口实例化 `new I()` | 接口实例化编译时错误 |
| 012 | 抽象类实例化 `new A()` | 抽象类实例化编译时错误 |

### runtime（验证运行时行为）

| # | 测试点 | 预期行为 |
|---|--------|---------|
| 013 | 构造器初始化 `new A(42)` | 运行时验证字段值为 42 |
| 014 | 方法链 `new A().method()` | 运行时方法调用结果为 99 |
| 015 | 无括号 `new A` | 运行时验证无括号创建字段值为 100 |
| 016 | 构造器异常 | 构造器 throw 抛出 Error |

## 文件命名规范

`EXP_07_15_YYY_{CATEGORY}_{DESCRIPTION}.ets`

- YYY: 001 起连续编号
- CATEGORY: PASS / FAIL / RUNTIME
- DESCRIPTION: 大写+下划线描述
