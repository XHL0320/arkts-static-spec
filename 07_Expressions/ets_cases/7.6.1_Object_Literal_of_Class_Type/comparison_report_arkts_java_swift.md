# 7.6.1 Object Literal of Class Type — 三语言对比报告

## 1. 概览

Object Literal of Class Type 定义了类类型上下文推断对象字面量类型的规则。三语言的行为差异显著，因为 Java 和 Swift 没有直接的对象字面量语法：

| 语言 | 定位 |
|------|------|
| **ArkTS** | 原生支持对象字面量语法：`{name: "Bob", age: 25}`，编译时进行字段名、类型、访问性、构造器、readonly 等多维度检查 |
| **Java** | 无对象字面量语法，使用构造器初始化或匿名类。字段访问检查在编译时进行 |
| **Swift** | 无对象字面量语法，使用构造器初始化。成员访问控制严格 |

## 2. 章节对应关系

| 规则 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 类上下文推断 | `{name: "Bob", age: 25}` | `new Person("Bob", 25)` | `Person(name: "Bob", age: 25)` |
| 访问控制 | 对象字面量中禁止 private/protected | new 表达式无此问题 | 构造器初始化无此问题 |
| 类型检查 | 编译时检查每个字段类型 | 编译时检查参数类型 | 编译时检查参数类型 |
| 跳过字段 | 有默认值/初始化器的字段可跳过 | 默认值通过无参构造器 | 默认值通过 `init()` |
| 构造器要求 | 必须有无参或全可选参数构造器 | 构造器可变 | 构造器可变 |
| readonly 限制 | 对象字面量不能设置 readonly | `final` 字段需构造器中设置 | `let` 属性需 `init` 中设置 |
| setter 支持 | setter 名称可作为对象字面量字段 | 无直接等价 | 无直接等价 |

## 3. 关键差异矩阵

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 对象字面量语法 | ✅ 原生支持 | ❌ 无（用构造器替代） | ❌ 无（用构造器替代） |
| 字段名检查 | ✅ 编译时（字段名必须存在） | ✅ 编译时（参数匹配） | ✅ 编译时（参数匹配） |
| private 字段 | ❌ 编译时错误 | ❌ 编译时错误（修饰符） | ❌ 编译时错误（修饰符） |
| protected 字段 | ❌ 编译时错误 | ✅ 子类可访问 | ✅ 同模块可访问（fileprivate） |
| 类型不匹配 | ❌ 编译时错误 | ❌ 编译时错误 | ❌ 编译时错误 |
| 只读字段限制 | ❌ 编译时错误 | ❌ 编译时错误（非构造器赋值） | ❌ 编译时错误 |
| setter 作为字段 | ✅ 通过对象字面量触发 setter | ❌ 无语法支持 | ❌ 无语法支持 |
| 字段跳过/默认值 | ✅ 原生支持 | ✅ 需构造器 | ✅ 需 init() |

## 4. 用例对照

### 4.1 对象字面量语法（ArkTS 独有）

ArkTS 是唯一支持对象字面量语法的语言：

| 语言 | 代码 | 说明 |
|------|------|------|
| ArkTS | `let p: Person = {name: "Bob", age: 25}` | 原生语法 |
| Java | `Person p = new Person("Bob", 25)` | 构造器替代 |
| Swift | `let p = Person(name: "Bob", age: 25)` | 构造器替代 |

### 4.2 setter 访问器支持（ArkTS 独有）

ArkTS 允许在对象字面量中使用 setter 名称，触发 setter 调用：

| 语言 | 代码 | 说明 |
|------|------|------|
| ArkTS | `let a: OK = {attr: 42}` | 触发 setter，编译通过 |
| Java | 无等价语法 | 需显式调用 setter: `obj.setAttr(42)` |
| Swift | 无等价语法 | 需直接赋值: `obj.attr = 42` |

### 4.3 字段跳过与默认值

| 语言 | 代码 | 说明 |
|------|------|------|
| ArkTS | `let f: Friend = {}` | 跳过所有有默认值的字段 |
| Java | `Friend f = new Friend()` | 默认构造器，所有字段默认初始化 |
| Swift | `let f = Friend()` | 默认 init()，所有属性默认值 |

### 4.4 protected 字段的差异

| 语言 | 对象字面量中 protected 字段 | 说明 |
|------|---------------------------|------|
| ArkTS | ❌ 编译时错误 | 同文件中也不可访问 |
| Java | ❌ 编译时错误（不同包） | 子类或同包可访问 |
| Swift | ⚠️ fileprivate 需要改级别 | 同文件中用 fileprivate 可访问 |

## 5. 三环境实测结果

由于对象字面量语法为 ArkTS 独有特性，Java 和 Swift 无直接等价语法，因此本节不进行三语言实测对比。相关 cross_lang_verify 目录包含各语言对应构造器方式的参考实现。

## 6. 综合评分

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 对象创建便捷性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| 字段访问控制 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 类型安全性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| readonly 保护 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| setter 集成 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |

## 7. 核心结论

1. **ArkTS 的对象字面量语法是独有优势**：Java 和 Swift 均无此语法，需通过构造器替代。
2. **访问控制实现一致**：private 字段在三种语言中均不可在对象字面量/构造器外直接设置。
3. **setter 支持是 ArkTS 独有特性**：直接通过对象字面量触发 setter，无需额外方法调用。
4. **默认值机制**：ArkTS 的字段跳过规则（有默认值/初始化器的字段可跳过）与 Java/Swift 的默认构造器语义等价。
5. **readonly 保护**：三种语言均对只读字段提供编译时保护。

## 8. ArkTS 设计建议

- 当前设计完善，无缺陷。
- 对象字面量语法显著提高了类实例创建的简洁性和可读性。
- 编译时多维检查（字段名、类型、访问性、构造器、readonly、setter）提供了全面的安全保证。
