# 7.6.1 Object Literal of Class Type - 测试设计思维导图

## 概述

从上下文推断出类类型 C 时，对象字面量的类型为 C。每个字段标识符必须命名类 C 的字段，且不能命名不可访问的成员（private/protected）。字段表达式的类型必须可赋值给字段类型，只有有默认值或显式初始化的字段可跳过。类必须有无参构造器（显式/隐式）或全可选参数构造器，不能显式设置 readonly 字段，setter 名称可用作对象字面量字段，getter-only 报错。

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 变量类型上下文 | `let p: Person = {name: "Bob", age: 25}` 变量类型上下文推断类类型 |
| 002 | 函数参数上下文 | `foo({name: "Alice", age: 18})` 函数参数上下文推断类类型 |
| 003 | 跳过所有默认值字段 | 所有字段有默认值/初始化器，`{}` 合法 |
| 004 | 跳过显式初始化字段 | 有显式初始化器的字段可跳过 |
| 005 | 隐式默认构造器 | 无显式构造器 → 隐式默认构造器可用 |
| 006 | 显式无参构造器 | 显式无参构造器可用 |
| 007 | 全可选参数构造器 | 全可选参数构造器可用 |
| 008 | 值类型字段默认值 | `number` 类型字段有默认值 0 可跳过 |
| 009 | setter 访问器 | setter 名称可用作对象字面量字段 |
| 010 | 部分字段提供 | 只有部分字段提供，其余使用默认值 |

### compile-fail

| # | 测试点 | 说明 |
|---|--------|------|
| 011 | private 字段 | 对象字面量中 private 字段 → 编译时错误 |
| 012 | protected 字段 | 对象字面量中 protected 字段 → 编译时错误 |
| 013 | 字段类型不兼容 | 字段表达式类型不兼容 → 编译时错误 |
| 014 | 必需字段未提供 | 无默认值/初始化器的字段未提供 → 编译时错误 |
| 015 | 无无参构造器 | 无无参构造器 → 编译时错误 |
| 016 | private 构造器 | private 构造器不可访问 → 编译时错误 |
| 017 | readonly 字段 | readonly 字段显式设置 → 编译时错误 |
| 018 | getter-only 访问器 | getter-only 访问器（无 setter）做字段 → 编译时错误 |

### runtime（验证实际计算值符合规则）

| # | 测试点 | 预期值 |
|---|--------|--------|
| 019 | 基本字段值 | name 和 age 字段值正确 |
| 020 | setter 访问器 | setter 通过对象字面量调用触发验证 |
| 021 | 默认字段值 | 跳过字段使用正确默认值 |

## 文件命名规范

`EXP_07_06_01_YYY_{CATEGORY}_{DESCRIPTION}.ets`

- YYY: 001 起连续编号
- CATEGORY: PASS / FAIL / RUNTIME
- DESCRIPTION: 大写+下划线描述

## 注释模板要求

每个文件必须包含 5 个 JSDoc tag：@id, @expect, @section, @design, @note
