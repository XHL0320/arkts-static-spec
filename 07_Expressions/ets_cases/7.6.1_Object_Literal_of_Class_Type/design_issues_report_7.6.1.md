# 7.6.1 Object Literal of Class Type — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: 对象字面量语法支持 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | 全部 compile-pass 用例 |
| **实测结果** | ArkTS 原生支持编译时对象字面量语法 |
| **错误信息** | 无 |

**描述**：ArkTS 是唯一原生支持对象字面量语法的语言：
- ArkTS：`let p: Person = {name: "Bob", age: 25}` 原生编译时对象字面量语法
- Java：需用构造器 `new Person("Bob", 25)`
- Swift：需用构造器 `Person(name: "Bob", age: 25)`

**跨语言对比**：

| 语言 | `let p: Person = {name: "Bob", age: 25}` |
|------|------------------------------------------|
| ArkTS | ✅ 原生编译时对象字面量语法 |
| Java | ❌ 需用构造器 `new Person("Bob", 25)` |
| Swift | ❌ 需用构造器 `Person(name: "Bob", age: 25)` |

**分类**：语言特性差异

**建议**：符合各自语言设计，无需调整。

### ID-02: setter 访问器集成 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_06_01_009_PASS_SETTER_ACCESSOR |
| **实测结果** | ArkTS 允许对象字面量通过 setter 名称触发 setter 调用 |
| **错误信息** | 无 |

**描述**：ArkTS 允许对象字面量通过 setter 名称触发 setter 调用。Java 和 Swift 不支持此模式，需显式调用 setter 方法或直接赋值。

**跨语言对比**：

| 语言 | setter 支持 |
|------|-----------|
| ArkTS | ✅ 通过对象字面量触发 setter |
| Java | ❌ 需显式调 setter 方法 |
| Swift | ❌ 需直接赋值 |

**分类**：语言特性差异

**建议**：符合 ArkTS 设计，无需调整。

### ID-03: 字段跳过与默认值机制 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_06_01_003_PASS_SKIP_FIELDS_DEFAULTS |
| **实测结果** | 有默认值/初始化器的字段可跳过 |
| **错误信息** | 无 |

**描述**：ArkTS 的"字段跳过"规则（有默认值/初始化器的字段可跳过）与 Java/Swift 的默认构造器/init() 语义等价，但 ArkTS 的对象字面量语法使部分赋值更简洁。

**跨语言对比**：

| 语言 | 字段跳过机制 |
|------|-------------|
| ArkTS | 有默认值/初始化器的字段在对象字面量中可跳过 |
| Java | 默认值通过无参构造器 |
| Swift | 默认值通过 `init()` |

**分类**：语法机制差异

**建议**：语义等价，无需调整。

### ID-04: protected 字段访问控制 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_06_01_012_FAIL_PROTECTED_FIELD |
| **实测结果** | 编译时错误（期望行为） |
| **错误信息** | 编译时错误 |

**描述**：ArkTS 在对象字面量上下文中对 protected 字段严格禁止，而 Java/Swift 在子类或同文件上下文中允许访问。差异源于对象字面量的类复合上下文限制。

**跨语言对比**：

| 语言 | 对象字面量中 protected 字段 |
|------|---------------------------|
| ArkTS | ❌ 编译时错误（同文件中也不可访问） |
| Java | ❌ 编译时错误（不同包），子类或同包可访问 |
| Swift | ⚠️ fileprivate 需改级别，同文件中用 fileprivate 可访问 |

**分类**：访问控制策略差异

**建议**：符合各自语言哲学，无需调整。
