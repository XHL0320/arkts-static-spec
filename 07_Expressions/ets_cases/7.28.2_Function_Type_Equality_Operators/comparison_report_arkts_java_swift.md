# 7.28.2 Function Type Equality Operators — 三语言对比报告

## 1. 概览

函数类型等值运算符定义了四个等值运算符在函数类型上的比较行为。

| 语言 | 函数类型比较 | 实例方法绑定 | 静态方法 | ===/!== |
|------|:-----------:|:-----------:|:--------:|:-------:|
| ArkTS | ✅ `foo == foo` | ✅ 绑定实例必须相同 | ✅ | ✅（与==同语义） |
| Java | ✅ 引用相等 | ✅ 方法引用对象 | ✅ | ❌ 无 |
| Swift | ❌ 编译时错误 | ❌ 不支持 | ❌ 不支持 | ❌（===仅类实例） |

## 2. 章节对应关系

| 功能点 | ArkTS 7.28.2 | Java JLS §15.12.5 | Swift |
|--------|-------------|-------------------|-------|
| 函数引用 == | `foo == foo` | 方法引用引用相等 | ❌ 不支持 |
| 实例方法比较 | `a.method == a.method` | `a::method == a::method` | ❌ 不支持 |
| 静态方法比较 | `A.method == A.method` | `A::method == A::method` | ❌ 不支持 |
| 不同实例方法 | false | false（不同对象） | ❌ 不支持 |
| 结果类型 | `boolean` | `boolean` | ❌ N/A |

## 3. 关键差异矩阵

| 差异点 | ArkTS | Java | Swift |
|--------|:-----:|:----:|:-----:|
| 函数/闭包引用比较 | ✅ 原生支持 | ✅ 间接支持 | ❌ 编译时错误 |
| 函数类型 === | ✅ 与==同语义 | ❌ 无 | ❌ 仅类实例 |
| 实例方法绑定比较 | ✅ 完整支持 | ✅ 参考相等 | ❌ 不支持 |
| Lambda 身份比较 | ✅ | ✅ | ❌ 不支持 |
| 底层设计 | 函数是引用类型 | 方法引用是对象 | 函数是值类型(名义) |

## 7. 核心结论

1. **ArkTS 函数类型等值比较** — 与 spec 完全一致，16/16 全部通过
2. **Java 方法引用比较** — 通过引用相等（reference equality）间接支持，语义基本一致
3. **Swift 不支持函数身份比较** — 这是语言设计层面的根本差异
4. **本质原因**：ArkTS 和 Java 的函数/方法引用是引用类型，Swift 函数是结构化类型

## 8. ArkTS 设计建议

1. **功能完整性** ✅ — 函数类型等值比较已完整实现
2. **建议保持** — 函数类型比较是 ArkTS 函数式编程支持的重要特性
