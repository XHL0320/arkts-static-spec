# 7.23.3 Remainder — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: 字面量整数取余除零未产生编译时错误 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_23_03_027_FAIL_REMAINDER_BY_ZERO_LITERAL |
| **实测结果** | 编译通过（期望编译时错误） |
| **错误信息** | 无错误 |

**描述**：根据 ArkTS spec 规则："if the divisor value of integer remainder operator is detected to be 0 during compilation, then a compile-time error occurs"。但编译器允许 `a % 0` 正常编译通过。与 7.23.2 中 `a / 0` 未检测的模式一致。

**跨语言对比**：

| 语言 | 代码 | 行为 |
|------|------|------|
| ArkTS (spec) | `a % 0` | 编译时错误 |
| ArkTS (实现) | `a % 0` | 编译通过 |
| Java | `a % 0` | 编译通过，运行时 ArithmeticException |
| Swift | `a % 0` | 编译通过，运行时 fatal error |

**分类**：D 类 — Spec 与实现不一致

**建议**：实现 spec 中字面量整数取余除零的编译时检测。

### ID-02: Bigint 字面量取余除零未检测（与除法不一致）⭐⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_23_03_028_FAIL_BIGINT_REMAINDER_BY_ZERO_LITERAL |
| **实测结果** | 编译通过（期望编译时错误） |
| **错误信息** | 无错误 |

**描述**：根据 spec，如果 bigint 除数为 0n 且编译时可检测，应报编译时错误。但编译器允许 `a % 0n` 正常编译通过。与 7.23.2 中 bigint 除法 `a / 0n` 被正确检测的行为不一致，说明除法和取余运算符的除零检测策略不同。

**跨语言对比**：

| 语言 | `a / 0n`（除法） | `a % 0n`（取余） |
|------|:----------------:|:----------------:|
| ArkTS (spec) | 编译时错误 | 编译时错误 |
| ArkTS (实现) | 编译时错误 | 编译通过 |
| Java | N/A | N/A |
| Swift | N/A | N/A |

**分类**：D 类 — Spec 与实现不一致

**建议**：统一除法和取余运算符的 int/bigint 除零检测策略。

### ID-03: 隐式类型提升差异（设计差异，非缺陷）

| 字段 | 值 |
|------|-----|
| **描述** | ArkTS 与 Java 一致：byte/short 自动提升到 int，运算结果取最大操作数类型。Swift 要求所有操作数类型一致或显式转换。 |
| **差异类型** | 符合 ArkTS spec 的语言设计差异 |

**跨语言对比**：

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 隐式类型提升 | byte/short→int, 最大类型 | 同 ArkTS | 禁止隐式提升 |
| bigint | 原生支持 | 无 | 无 |

**分类**：符合 ArkTS spec 的语言设计差异

### ID-04: 浮点取余语义差异（Swift 使用方法调用）

| 字段 | 值 |
|------|-----|
| **描述** | ArkTS 和 Java 的 `%` 运算符在浮点数上使用 truncating/fmod 风格。Swift 需要使用显式的 `.truncatingRemainder(dividingBy:)` 方法。 |
| **差异类型** | 符合 ArkTS spec 的语言设计差异 |

**跨语言对比**：

| 语言 | 浮点取余代码 |
|------|-------------|
| ArkTS | `10.5 % 3.0` |
| Java | `10.5 % 3.0` |
| Swift | `10.5.truncatingRemainder(dividingBy: 3.0)` |

**分类**：符合 ArkTS spec 的语言设计差异

### ID-05: INT_MIN % -1 行为差异

| 字段 | 值 |
|------|-----|
| **描述** | ArkTS 和 Java 结果为 0。Swift 中 INT_MIN % -1 因内部除法的溢出导致运行 trap。 |
| **差异类型** | 符合 ArkTS spec 的语言设计差异 |

**跨语言对比**：

| 语言 | INT_MIN % -1 | 说明 |
|------|:-----------:|------|
| ArkTS | 0 | 正确返回 |
| Java | 0 | 同 ArkTS |
| Swift | Trap! | 内部除法溢出 trap |

**分类**：符合 ArkTS spec 的语言设计差异
