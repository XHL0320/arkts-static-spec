# 7.2.3 Order of Expression Evaluation — ArkTS 与 Java/Swift/TS 行为差异及规范一致性报告

## 设计问题及差异清单

### ID-01: Swift 赋值返回 Void ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_02_03_005, 019 |
| **实测结果** | ArkTS 编译通过并正确执行；Java 编译通过并正确执行；Swift 编译错误 |
| **错误信息** | Swift: "Cannot assign to value: function returns Void" |

**描述**：Swift 中赋值运算符返回 Void（不返回值），因此不支持链式赋值 `a = b = c = value`。ArkTS 和 Java 的赋值运算符返回被赋的值，支持链式写法。

**跨语言对比**：

| 语言 | 代码 | 行为 |
|------|------|------|
| ArkTS | `a.val = b.val = "x"` | 编译通过，结果 "x" |
| Java | `a.val = b.val = "x"` | 编译通过，结果 "x" |
| Swift | `a.val = b.val = "x"` | 编译错误 |

**分类**：符合各自语言设计哲学的差异

**建议**：无需修改，ArkTS 行为与 Java 一致更符合主流开发者预期

### ID-02: 短路求值三语言完全一致 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_02_03_003, 004, 011, 017, 018, 022 |
| **实测结果** | 三语言行为完全一致，全部通过 |
| **错误信息** | 无 |

**描述**：ArkTS、Java、Swift 在 `&&` 和 `||` 的短路求值语义上完全一致：从左到右求值，`&&` 在左为 false 时跳过右，`||` 在左为 true 时跳过右。链式逻辑三层短路行为也完全一致。

**跨语言对比**：

| 语言 | && 短路 | \|\| 短路 | 链式短路 |
|------|---------|-----------|---------|
| ArkTS | 左 false 跳过右 | 左 true 跳过右 | 行为一致 |
| Java | 左 false 跳过右 | 左 true 跳过右 | 行为一致 |
| Swift | 左 false 跳过右 | 左 true 跳过右 | 行为一致 |

**分类**：三语言行为一致

**建议**：无需修改

### ID-03: 参数异常传播 — 左侧异常跳过右侧 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_02_03_021 |
| **实测结果** | ArkTS 和 Java 异常可捕获，右侧参数被跳过；Swift 除零触发 fatal error |
| **错误信息** | ArkTS: ArithmeticError; Java: ArithmeticException; Swift: fatal error |

**描述**：ArkTS 和 Java 在函数参数求值上行为一致：参数从左到右求值，左侧参数异常完成时右侧参数不被求值，异常可被 try-catch 捕获。Swift 因除零触发 fatal error（不可恢复、不可捕获），无法直接比较。

**跨语言对比**：

| 语言 | 左侧参数抛异常 | 右侧参数是否求值 | 异常可捕获 |
|------|---------------|------------------|-----------|
| ArkTS | ArithmeticError | 跳过 | 是 |
| Java | ArithmeticException | 跳过 | 是 |
| Swift | fatal error | 跳过 | 否（不可恢复） |

**分类**：ArkTS 与 Java 一致，Swift 因 fatal error 不可比

**建议**：无需修改

### ID-04: 浮点数非结合性三语言通用 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_02_03_024 |
| **实测结果** | 三语言均观察到 (a+b)+c != a+(b+c) 的浮点非结合行为 |
| **错误信息** | 无 |

**描述**：三语言均观察到 `(a+b)+c != a+(b+c)` 的浮点非结合行为。ArkTS 采用 IEEE 754 标准浮点运算，与 Java/Swift 一致。

**跨语言对比**：

| 语言 | 浮点运算标准 | (a+b)+c != a+(b+c) |
|------|-------------|-------------------|
| ArkTS | IEEE 754 | 观察得到 |
| Java | IEEE 754 | 观察得到 |
| Swift | IEEE 754 | 观察得到 |

**分类**：三语言行为一致（IEEE 754 标准结果）

**建议**：无需修改

### ID-05: compile-fail nullish 混合检查一致 ⭐

| 字段 | 值 |
|------|-----|
| **复现用例** | EXP_07_02_03_013, 014 |

| ArkTS | 编译时错误（spec 要求） |
| Java | 编译通过（无此限制） |
| Swift | 编译通过（无此限制） |

**分类**：符合 ArkTS spec 的设计

**建议**：无需修改，保持 ArkTS 特有的安全性设计
