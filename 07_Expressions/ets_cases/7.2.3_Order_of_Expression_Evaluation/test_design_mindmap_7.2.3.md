# 7.2.3 Order of Expression Evaluation - 测试设计思维导图

## 概述

本节定义表达式子表达式的求值顺序规则，包括左到右求值、三元条件优先、短路求值（&& 左为 false 时跳过右、|| 左为 true 时跳过右）、赋值右结合（LHS/MID/RHS 子表达式从左到右求值，然后从右到左赋值）、括号/优先级（显式括号和隐式运算符优先级决定求值顺序）、浮点数非结合性（浮点运算不可随意重排，不满足结合律）以及函数参数左到右求值（左侧异常完成则跳过右侧）。

## 注释模板要求

每个测试文件需包含以下 JSDoc 标签：

```typescript
/**
 * @id EXP_07_02_03_YYY_{CATEGORY}_{DESC}
 * @expect compile-pass | compile-fail | runtime
 * @section 7.2.3
 * @design <测试内容描述>
 * @note 正向用例：验证该特性编译通过的正确用法
 *       反向用例：验证该场景应产生编译时错误
 *       运行时用例：验证该场景的运行时行为
 */
```

Runtime 用例必须有 `main()` 函数并使用断言验证：

```typescript
function main(): void {
  if (actualResult != expectedResult) {
    throw new Error("assertion failed: " + actualResult)
  }
  console.log("verified")
}
```
