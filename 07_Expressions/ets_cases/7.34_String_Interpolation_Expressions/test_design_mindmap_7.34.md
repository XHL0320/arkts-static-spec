# 7.34 String Interpolation Expressions - 测试设计思维导图

## 概述

验证 ArkTS 中字符串插值表达式（String Interpolation Expression）的正确性：使用反引号（backtick）界定的多行字符串字面量，通过 `${expression}` 语法嵌入表达式的求值结果。所有嵌入表达式求值后，以其字符串形式替换原 `${}` 占位，结果类型始终为 `string`。非 string 类型的嵌入表达式与字符串拼接运算符（`+`）相同方式隐式转换，嵌入表达式中可包含嵌套的多行字符串（反引号）。

### Spec 规则

| 规则 | 说明 |
|------|------|
| 语法 | 反引号界定 `` ` ` ` ` ``，可包含任意数量 `${expression}` |
| 结果类型 | 始终为 `string` |
| 表达式替换 | 每个嵌入表达式求值后，以其字符串形式替换原 `${}` 占位 |
| 隐式转换 | 非 string 类型的嵌入表达式与字符串拼接运算符（`+`）相同方式隐式转换 |
| 嵌套 | 嵌入表达式中可包含嵌套的多行字符串（反引号） |
| 等价 | 字符串插值可用字符串拼接运算符重写 |

## 三级测试点设计

### compile-pass（验证编译通过 + 语义正确）

| # | 测试点 | 说明 |
|---|--------|------|
| 001 | 基本类型变量插值 | string/int/long/float/double/boolean 六种基本类型作为嵌入表达式 |
| 002 | 算术表达式插值 | `${a + b}`, `${a * b}`, `${a - b}`, `${a / b}` 等算术运算嵌入 |
| 003 | 多个嵌入表达式 | 连续多个 `${}` 及混合字符串和数字的嵌入 |
| 004 | 方法/函数调用嵌入 | `${foo()}`, `"hello".length()`, `${Math.abs(-5)}` 等 |
| 005 | 布尔/null/undefined 插值 | `${true}`, `${false}`, `${undefined}`, `${x ?? "default"}` |
| 006 | 字段/数组元素插值 | `${obj.x}`, `${arr[0]}` |
| 007 | 嵌套反引号插值 | `` `outer ${`inner ${x}`}` `` 及双重嵌套 |
| 008 | 纯多行反引号（无插值） | 普通反引号字符串不含 `${}`，含换行符 |
| 009 | 与字符串拼接等价 | 验证插值表达式与 `+` 拼接等价 |
| 010 | 复杂表达式 | 三元表达式、substring 调用等复杂场景的插值 |

### compile-fail

| # | 测试点 | 说明 |
|---|--------|------|
| 011 | 嵌入表达式类型错误 | `${x.substring(0)}` 当 x 为 int 类型应产生编译时错误 |
| 012 | 插值结果赋给非 string | `let y: int = \`${x}\`` 应产生编译时类型错误 |

### runtime（验证实际计算值符合字符串插值规则）

| # | 测试点 | 预期值 |
|---|--------|--------|
| 013 | 变量/算术/多表达式/布尔/方法/字段/数组/拼接等价/无插值 | 12 个断言验证插值结果字符串正确性 |
