## 一、行为差异与规范一致性概览

### 观察点 H：隐式泛型类型推断 ⭐⭐⭐ HIGH

**用例：** GEN_05_02_301~315 系列

**实际行为：**
```typescript
// ArkTS 支持隐式泛型类型推断
let b1 = new Box(42)              // 推断 Box<int>
let r1 = foo(42)                  // 推断 foo<int>()
let r2 = identity("hello")        // 推断 identity<string>()

// 无法推断时编译拒绝
let b2 = new Box()                 // ❌ 无法推断类型
```

**对比：**
| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 函数泛型推断 | ✅ 支持 | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| 多参数推断 | ✅ 支持 | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| 方法默认参数推断 | ✅ 支持 | ✅ 部分支持 | ✅ 支持 | ✅ 支持 |
| 无法推断类型 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 |
| 无上下文推断 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 |
| 默认参数顺序错误 | ❌ 编译拒绝 | N/A | ❌ 编译拒绝 | ❌ 编译拒绝 |

**影响：** ArkTS 的类型推断系统与 TypeScript/Java/Swift 基本一致。

**评价：** 无问题，设计合理。
