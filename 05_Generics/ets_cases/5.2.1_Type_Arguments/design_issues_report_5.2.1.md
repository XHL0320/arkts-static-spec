## 一、行为差异与规范一致性概览

### 观察点 F：丰富的类型参数支持 ⭐⭐⭐ HIGH

**用例：** GEN_05_02_101~115 系列

**实际行为：**
```typescript
// ArkTS 支持多种类型参数
let b1: Box<int> = new Box<int>()              // 数字类型参数
let b2: Box<A | B> = new Box<A | B>()          // 联合类型参数
let b3: Box<int[]> = new Box<int[]>()          // 数组类型参数
let b4: Box<[int, string]> = new Box<[int, string]>()  // 元组类型参数
let b5: Box<() => void> = new Box<() => void>()  // 函数类型参数
```

**对比：**
| 维度 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 数字类型参数 | ✅ `Box<int>` | ✅ `Box<Integer>` | ✅ `Box<Int>` | ✅ `Box<number>` |
| 联合类型参数 | ✅ `Box<A \| B>` | ❌ 不支持联合 | ❌ 不支持联合 | ✅ `Box<A \| B>` |
| 数组类型参数 | ✅ `Box<int[]>` | ✅ `Box<int[]>` | ✅ `Box<[Int]>` | ✅ `Box<number[]>` |
| 元组类型参数 | ✅ `Box<[int, string]>` | ❌ 不支持 | ✅ `Box<(Int, String)>` | ✅ `Box<[number, string]>` |
| 函数类型参数 | ✅ `Box<() => void>` | ❌ 不支持（函数类型非一等） | ✅ `Box<() -> Void>` | ✅ `Box<() => void>` |
| 无效类型参数 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 | ❌ 编译拒绝 |

**影响：** ArkTS 的类型参数系统比 Java 更丰富，与 TypeScript 高度一致。

**评价：** 无问题，设计合理。
