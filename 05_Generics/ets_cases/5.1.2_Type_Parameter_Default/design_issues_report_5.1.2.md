## 一、行为差异与规范一致性概览

### 观察点 C：类型参数默认值支持 ⭐⭐⭐ HIGH

**用例：** GEN_05_01_201~215 系列

**实际行为：**
```typescript
// ArkTS 支持类型参数默认值
interface Box<T = string> {}
interface Pair<T = string, U = number> {}

// 默认后不可有无默认
interface Bad<A = int, B> {}  // ❌ 编译拒绝

// 默认值前向引用
interface Bad<T = U, U = string> {}  // ❌ 编译拒绝
```

**对比：**
| 语言 | 类型参数默认值 | 多默认参数 | 函数默认类型参数 | 默认后不可有无默认 | 默认值前向引用 |
|------|-------------|-----------|-----------------|---------------|-------------|
| ArkTS | ✅ `T = U` | ✅ `T = A, U = B` | ✅ 支持 | ❌ 编译拒绝 | ❌ 编译拒绝 |
| Java | ❌ 不支持 | ❌ 不支持 | ❌ 不支持 | N/A | N/A |
| Swift | ✅ `T = U` | ✅ | ✅ 支持 | ❌ 编译拒绝 | ❌ 编译拒绝 |
| TypeScript | ✅ `T = U` | ✅ | ✅ 支持 | ❌ 编译拒绝 | ❌ 编译拒绝 |

**影响：** ArkTS 的默认值系统与 Swift/TypeScript 一致，比 Java 更灵活。

**评价：** 无问题，设计合理。
