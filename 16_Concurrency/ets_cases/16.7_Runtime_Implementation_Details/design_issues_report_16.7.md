## 3. 详细分析

### 3.1 问题 1-3: 整体覆盖不足

- **问题描述**：16.7 仅有 2 个基编译用例，无反向和运行时用例，大量调度规则未覆盖
- **建议方案**：后续迭代逐步增补

## 4. 说明

- 调度规则 spec 定义的优先级（suspendable > other）和 FIFO 顺序均为**运行时行为**，无法通过 compile-pass/fail 测试验证
- compile-pass 仅能验证调度相关 API 的语法正确性
- spec_original.md 已从根级 spec 提取创建

## 5. 改进建议

- 运行时调度规则验证：使用 ark VM 执行多个 job 并验证执行顺序
- 优先级规则：高优先级 suspendable job 先于低优先级普通 job
- 同优先级 FIFO：按提交顺序执行
- worker 线程管理：创建/删除场景（依赖于 EAWorker stdlib）
