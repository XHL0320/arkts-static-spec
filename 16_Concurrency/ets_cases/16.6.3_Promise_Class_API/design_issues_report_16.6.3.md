# 16.6.3 Promise Class API - 设计问题报告

## 1. 概述

当前覆盖 Promise 类 then、catch、finally 三个核心方法的编译验证。链式调用、值传递、异常传播等需运行时验证。

## 2. 设计问题清单

| # | 问题类型 | 问题描述 | 建议方案 | 优先级 |
|---|---------|---------|---------|:------:|
| 1 | 覆盖不足 | 缺少 runtime 用例验证 then/catch 回调实际调用 | 使用 ark VM 执行 Promise 链 | 高 |
| 2 | 覆盖不足 | Promise 链式 then 调用未测试 | 增补 then 链的编译用例 | 中 |
| 3 | 覆盖不足 | finally 中抛出异常的行为未覆盖 | 增补 finally 异常传播用例 | 低 |

## 3. 改进建议

- runtime 测试：验证 then → catch → finally 完整链的执行顺序
- 编译期：验证 then 返回新 Promise 的类型兼容性
