# 16.7.1 Scheduling Rules - 设计问题报告

## 1. 概述

当前覆盖调度规则基本语法的编译验证。调度规则的实际运行时语义（优先级、抢占、公平性）需运行时验证。调度规则的参数类型约束也需补充。

## 2. 设计问题清单

| # | 问题类型 | 问题描述 | 建议方案 | 优先级 |
|---|---------|---------|---------|:------:|
| 1 | 覆盖不足 | 缺少 runtime 用例验证调度策略实际生效 | 使用 ark VM 执行调度规则用例 | 高 |
| 2 | 覆盖不足 | 调度规则参数类型和范围未覆盖 | 增补不同参数类型的编译用例 | 中 |
| 3 | 覆盖不足 | 调度规则与 launch/async 的组合未测试 | 增补调度规则+ async 函数的场景 | 中 |

## 3. 改进建议

- runtime 测试：验证设置调度策略后 job 按预期顺序执行
- 编译期：验证非法调度参数（越界、类型不匹配）是否应报错
