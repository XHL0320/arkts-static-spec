## 一、新发现的设计问题（基于规范分析）

未发现设计问题。所有行为均符合 ArkTS 规范第 8.7 节。

以下方面已验证与规范一致：

1. **语法正确性**：`whileStatement`（`'while' '(' expression ')' statement`）与 `doStatement`（`'do' statement 'while' '(' expression ')' ';'`）均严格遵循规范定义的语法，无任何偏差。

2. **条件类型强制检查**：规范要求表达式类型必须为 `boolean` 或扩展条件表达式中提及的类型。测试用例确认：
   - 布尔变量、布尔字面量、比较表达式（`<`、`>`、`===` 等）以及逻辑表达式（`&&`、`||`、`!`）均可成功编译。
   - 数字字面量、字符串字面量以及数字变量出现在条件位置时会产生编译时错误，符合规范要求。

3. **语义区分**：规范规定 `while` 先评估条件，而 `do` 先执行循环体。运行时测试用例确认：
   - `while(false)` 循环体执行零次。
   - `do { ... } while (false)` 循环体恰好执行一次。
   - 在初始条件相同时，两者执行相同次数的迭代。

4. **空循环体支持**：规范语法允许任意 `statement` 作为循环体。空块 `{}` 和空语句 `;` 均为合法形式，已由编译通过用例确认。

5. **嵌套循环**：while 与 do-while 可自由嵌套，已由编译通过用例 case 005 确认。
