## 四、跨语言对比结论

### 观察 A：break 的标签语义与 Java 一致，对 Swift 开发者需注意区别

**说明：** ArkTS 规范 8.10 节的 break 标签规则与 Java JLS SE21 第 14.15 节几乎完全相同：

- 无标签 break 跳出最内层 while/do/for/for-of/switch
- 带标签 break 跳出包含该标签的封闭语句
- 标签必须指向一个循环语句（loopStatement）或 switch 语句

**与 Swift 的微妙区别：**

| 维度 | ArkTS / Java | Swift |
|------|-------------|------|
| switch 中默认 fallthrough | 需要 break 防止 fallthrough | 默认不 fallthrough（无需 break）|
| 显式 fallthrough | 不适用 | 使用 `fallthrough` 关键字 |
| break 标签只能指向循环/switch | 是（编译时检查）| 是 |

**评价：** 行为完全符合规范，无设计问题。Swift 开发者需注意 switch 中 break 的语义差异。ArkTS 的 break 语句设计严谨，规则清晰，与 Java 语义完全一致。
