## 四、跨语言对比结论

### 对比点：delete 运算符支持情况

ArkTS 规范 8.2 节未明确列出禁用的运算符，但测试验证了 `delete` 作为表达式语句时报编译错误（用例 006）。

| 语言 | delete 运算符状态 |
|------|------------------|
| ArkTS | ❌ 不支持（编译错误） |
| Java | ❌ 无 delete 运算符（对象由 GC 管理） |
| Swift | ❌ 无 delete 运算符（ARC 管理） |

**评价：** ArkTS 禁用 `delete` 是合理设计，与 Java/Swift 一致。JavaScript 中的 `delete` 在 ArkTS 静态类型系统中没有存在意义——ArkTS 采用自动内存管理，不需要也不应该允许手动删除对象属性。
