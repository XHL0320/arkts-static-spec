# 15.1.7 Specifics of Type Parameters - Design Issues Report

## 一、编译器实现待完善 (Compiler Implementation Gaps)

### 1.1 类型参数作为返回值类型的完整支持
- **问题描述**: ArkTS 规范支持类型参数作为返回值类型，但当前编译器可能未完全支持所有场景
- **用例**: SEM_15_01_07_001_PASS_TYPE_PARAM_RETURN.ets
- **期望行为**: 泛型函数返回类型参数类型（如 identity<int>(42) 返回 int 类型）
- **当前状态**: 待验证
- **优先级**: 高

### 1.2 类型参数约束的完整支持
- **问题描述**: ArkTS 规范支持类型参数约束（extends 约束），但当前编译器可能未完全支持所有约束场景
- **用例**: SEM_15_01_07_002_PASS_TYPE_PARAM_CONSTRAINT.ets
- **期望行为**: extends 约束的类型参数可调用约束方法（如 T extends HasName，可访问 .name 属性）
- **当前状态**: 待验证
- **优先级**: 高

### 1.3 类型参数作为 LHS 时不提供推断信息的完整支持
- **问题描述**: ArkTS 规范规定类型参数作为 LHS 时不提供推断信息，但当前编译器可能未完全实现此特性
- **用例**: SEM_15_01_012_FAIL_TYPE_PARAM_LHS_INFERENCE.ets
- **期望行为**: 当 LHS 为类型参数 T 时，RHS 应按独立表达式推断，不应推断为 T
- **当前状态**: 待验证
- **优先级**: 中

## 二、与业界静态语言差异点 (Differences from Industry Static Languages)

### 2.1 类型参数作为 LHS 时的类型推断
- **ArkTS 行为**: 类型参数作为 LHS 时不提供推断信息（按独立表达式推断 RHS）
- **Java 行为**: 类似，类型参数作为 LHS 时不提供推断信息
- **Swift 行为**: 支持上下文推断，类型参数作为 LHS 时可以提供推断信息
- **TypeScript 行为**: 支持上下文推断，类型参数作为 LHS 时可以提供推断信息
- **差异分析**: ArkTS 和 Java 的类型推断能力较弱，Swift 和 TypeScript 的上下文推断更强大
- **设计意图**: 保持与 Java 一致的简单类型推断语义

### 2.2 类型参数约束的使用
- **ArkTS 行为**: 支持 extends 约束，约束的类型参数可调用约束方法
- **Java 行为**: 类似，支持 extends 约束
- **Swift 行为**: 支持协议约束（protocol），功能类似
- **TypeScript 行为**: 支持 extends 约束，功能类似
- **差异分析**: ArkTS 与 Java/TypeScript 一致，Swift 使用协议约束
- **设计意图**: 保持与主流静态语言一致的泛型约束语义

## 三、Spec措辞待澄清 (Spec Wording Clarification Needed)

### 3.1 类型参数作为 LHS 时不提供推断信息的精确定义
- **问题**: Spec 中"类型参数作为 LHS 时不提供推断信息"的规则不够明确，具体如何影响类型推断？
- **建议**: 明确类型参数作为 LHS 时的类型推断规则，并给出示例
- **影响范围**: 15.1.7
- **优先级**: 中

### 3.2 类型参数约束的详细规则
- **问题**: Spec 中类型参数约束的规则不够明确，哪些约束是允许的？约束如何用于类型推断？
- **建议**: 补充类型参数约束的详细规则，包括多个约束、约束的继承等
- **影响范围**: 15.1.7, 5.1
- **优先级**: 中

## 四、测试覆盖度分析 (Test Coverage Analysis)

### 4.1 已覆盖场景
- ✅ 类型参数作为返回值类型（泛型函数返回类型参数类型）
- ✅ 类型参数约束（extends约束的类型参数可调用约束方法）
- ✅ 类型参数作为LHS时不提供推断信息
- ✅ 类型参数运行时行为（泛型函数运行时派发正确）

### 4.2 待补充场景
- ⚠️ 多个类型参数的场景
- ⚠️ 类型参数约束的复杂场景（多个约束、约束的继承等）
- ⚠️ 类型参数作为方法参数的场景
- ⚠️ 类型参数作为字段类型的场景

| 类型参数作为方法参数的场景 | 高 | 常见泛型编程场景 |
