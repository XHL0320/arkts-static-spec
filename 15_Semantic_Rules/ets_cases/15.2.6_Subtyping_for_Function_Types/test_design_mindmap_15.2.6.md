# 15.2.6 Subtyping for Function Types - 测试设计思维导图

## 测试范围

**规范章节**: 15.2.6 Subtyping for Function Types

**测试目标**: 验证 ArkTS 中函数类型的子类型关系，包括参数类型逆变、返回值类型协变等。

**关键概念**:
- 函数类型子类型: `(A) => B <: (C) => D` 当且仅当 `C <: A`（参数逆变）且 `B <: D`（返回值协变）
- 参数逆变: 参数类型必须反过来（父类参数可以接受子类参数）
- 返回值协变: 返回值类型必须一致或派生（子类返回值可以赋给父类变量）

## 测试设计思路

### 1. 函数类型子类型（正向）
- **测试点**: 参数逆变 + 返回值协变
- **测试代码**: `(Animal) => Dog <: (Dog) => Animal`
- **预期结果**: 编译通过，运行时正常

### 2. 参数协变违规（反向）
- **测试点**: 参数协变违规拒绝
- **测试代码**: `(Dog) => void` 不可用作 `(Animal) => void`
- **预期结果**: 编译失败

### 3. 运行时行为（运行时）
- **测试点**: 验证函数类型子类型在运行时的行为
- **测试代码**: 传递函数作为参数
- **预期结果**: 运行时行为符合预期


## 最新设计要点

从章节思维导图同步的最新测试设计点：

- parameter contravariance
- return type covariance
- function type assignment compatibility
