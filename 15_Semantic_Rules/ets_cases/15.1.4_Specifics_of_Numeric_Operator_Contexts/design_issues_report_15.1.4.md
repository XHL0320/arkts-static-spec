# 15.1.4 Specifics of Numeric Operator Contexts - Design Issues Report

## 一、编译器实现待完善 (Compiler Implementation Gaps)

### 1.1 数值运算符操作数拓宽的完整支持
- **问题描述**: ArkTS 规范规定数值运算符操作数应拓宽到更大类型（最低 int），但当前编译器可能未完全支持所有拓宽场景
- **用例**: SEM_15_01_010_PASS_NUMERIC_OPERATOR_WIDEN.ets
- **期望行为**: byte+short→int, int+double→double
- **当前状态**: 待验证
- **优先级**: 高

### 1.2 byte/short 自增不自宽的完整支持
- **问题描述**: ArkTS 规范规定 byte/short 自增不自宽，但当前编译器可能未完全支持此特性
- **用例**: SEM_15_01_021_PASS_BYTE_SHORT_INC.ets
- **期望行为**: byte/short 自增后保持原类型
- **当前状态**: 待验证
- **优先级**: 中

### 1.3 混合数值类型运算拓宽的完整支持
- **问题描述**: ArkTS 规范支持混合数值类型运算拓宽（如 int + long → long），但当前编译器可能未完全支持所有拓宽场景
- **用例**: SEM_15_01_022_PASS_MIXED_NUMERIC.ets
- **期望行为**: 混合数值类型运算应拓宽到更大类型
- **当前状态**: 待验证
- **优先级**: 中

## 二、与业界静态语言差异点 (Differences from Industry Static Languages)

### 2.1 数值运算符操作数拓宽一致性
- **ArkTS 行为**: 数值运算符操作数拓宽到更大类型（最低 int）
- **Java 行为**: 类似，binary numeric promotion（最低 int）
- **Swift 行为**: 不支持隐式拓宽，需要显式转换
- **TypeScript 行为**: 所有数值都是 number 类型，不存在拓宽问题
- **差异分析**: ArkTS 与 Java 一致，Swift 需要显式转换，TypeScript 无此问题
- **设计意图**: 保持与 Java 一致的数值运算语义

### 2.2 byte/short 自增不自宽
- **ArkTS 行为**: byte/short 自增不自宽，保持原类型
- **Java 行为**: 类似，byte/short 自增结果仍为 byte/short
- **Swift 行为**: 不支持 byte/short 类型
- **TypeScript 行为**: 所有数值都是 number 类型，不存在此问题
- **差异分析**: ArkTS 与 Java 一致
- **设计意图**: 保持与 Java 一致的数值运算语义

### 2.3 布尔值数值运算拒绝
- **ArkTS 行为**: 布尔值不能用于数值运算（boolean + int 无效）
- **Java 行为**: 类似，布尔值不能用于数值运算
- **Swift 行为**: 类似，布尔值不能用于数值运算
- **TypeScript 行为**: 类似，布尔值不能用于数值运算
- **差异分析**: 所有静态语言都拒绝布尔值数值运算
- **设计意图**: 类型安全

## 三、Spec措辞待澄清 (Spec Wording Clarification Needed)

### 3.1 数值运算符操作数拓宽的精确定义
- **问题**: Spec 中"数值运算符操作数拓宽到更大类型（最低 int）"的规则不够明确，哪些运算符会触发拓宽？
- **建议**: 明确列举所有会触发拓宽的运算符（+, -, *, /, %, etc.）
- **影响范围**: 15.1.4 整节
- **优先级**: 中

### 3.2 混合数值类型运算拓宽的优先级
- **问题**: 当有多个操作数时，拓宽的优先级和规则不够明确（如 int + long + double）
- **建议**: 补充混合数值类型运算拓宽的详细规则
- **影响范围**: 15.1.4
- **优先级**: 中

## 四、测试覆盖度分析 (Test Coverage Analysis)

### 4.1 已覆盖场景
- ✅ 数值运算符操作数拓宽（int+int→int, int+double→double, byte+short→int）
- ✅ byte/short 自增不自宽
- ✅ 混合数值类型运算拓宽（int+long→long）
- ✅ 非数值运算符不可用于数值（boolean+int无效）
- ✅ 数值运算符运行时行为

### 4.2 待补充场景
- ⚠️ 各种数值运算符的拓宽场景（+， -， *， /， %）
- ⚠️ 复合赋值运算符的数值拓宽（+=， -=， etc.）
- ⚠️ 位运算符的数值拓宽（&, |, ^, ~）
- ⚠️ 移位运算符的数值拓宽（<<, >>, >>>）
