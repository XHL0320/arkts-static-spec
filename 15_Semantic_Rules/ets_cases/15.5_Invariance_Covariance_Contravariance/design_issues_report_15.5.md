## 二、与 Java/Swift/TypeScript 行为差异

### 2.1 变型语义对比
| 方面 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 泛型不变性 | ✅ 默认不变 | ✅ 默认不变 | ✅ 默认不变 | 结构等价 |
| 返回值协变 | ✅ override 支持 | ✅ @Override 支持 | ✅ override 支持 | ✅ 支持 |
| 参数逆变 | ✅ 函数类型支持 | ✅ SAM 接口支持 | ✅ 闭包支持 | ✅ 支持 |
| 声明点变型 | ❌ 不支持 | ❌ 不支持（使用点变型） | ✅ 支持 | ❌ 不支持 |
| 使用点变型 | ❌ 不支持 | ✅ 通配符 | N/A | N/A |

### 2.2 关键差异点
1. **ArkTS vs Java**：
   - 两者都默认泛型不变，支持返回值协变
   - Java 支持使用点变型（通配符 ? extends, ? super），ArkTS 不支持
   - ArkTS 函数类型参数支持逆变，Java 通过 SAM 接口实现类似功能

2. **ArkTS vs Swift**：
   - Swift 支持声明点变型（inout 参数），ArkTS 不支持
   - 两者都支持返回值协变和参数逆变
   - Swift 闭包天然支持函数类型逆变

3. **ArkTS vs TypeScript**：
   - TypeScript 是结构类型系统，泛型变型基于结构等价
   - ArkTS 是名义类型系统，泛型不变
   - 两者都支持返回值协变和参数逆变

## 三、规范措辞待澄清

### 3.1 声明点变型 vs 使用点变型
- **问题**：规范未明确说明 ArkTS 是否支持声明点变型（Declaration-site Variance）或使用点变型（Use-site Variance）
- **建议**：明确说明 ArkTS 的变型规则，是否支持类似 Java 的通配符或 Swift 的声明点变型
- **当前实现**：ArkTS 泛型默认不变，不支持声明点变型或使用点变型

### 3.2 函数类型变型规则
- **问题**：规范未明确说明函数类型的变型规则（参数类型逆变，返回类型协变）
- **建议**：明确说明函数类型的变型规则，与现代语言设计一致
- **当前实现**：ArkTS 函数类型支持参数逆变和返回类型协变

### 3.3 数组协变问题
- **问题**：规范未明确说明数组类型的变型规则（协变、不变、还是逆变）
- **建议**：明确说明数组类型的变型规则，避免 Java 数组协变的类型安全问题
- **当前实现**：ArkTS 数组类型不变（与 Java 不同，Java 数组是协变的）
