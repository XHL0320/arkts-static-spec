## 二、与 Java/Swift/TypeScript 行为差异

### 2.1 类型推断语义对比
| 方面 | ArkTS | Java | Swift | TypeScript |
|------|-------|------|-------|------------|
| 变量类型推断 | ✅ `let x = 42` | ❌ 需要显式类型或 var | ✅ `let x = 42` | ✅ `let x = 42` |
| 函数返回类型推断 | ✅ 支持 | ❌ 需要显式返回类型 | ✅ 支持 | ✅ 支持 |
| 泛型类型推断 | ✅ 支持 | ✅ 支持（Java 10+） | ✅ 支持 | ✅ 支持 |
| 上下文类型推断 | ✅ 支持 | ❌ 有限支持 | ✅ 支持 | ✅ 支持 |

### 2.2 关键差异点
1. **ArkTS vs Java**：
   - ArkTS 支持变量类型推断（`let x = 42`），Java 需要 var（Java 10+）
   - 两者都支持泛型类型推断和返回类型推断

2. **ArkTS vs Swift**：
   - 两者都支持变量类型推断、返回类型推断、泛型类型推断
   - Swift 的类型推断更强大（如闭包参数推断）

3. **ArkTS vs TypeScript**：
   - 两者都支持变量类型推断、返回类型推断、泛型类型推断
   - TypeScript 的类型推断更强大（如上下文类型推断）
