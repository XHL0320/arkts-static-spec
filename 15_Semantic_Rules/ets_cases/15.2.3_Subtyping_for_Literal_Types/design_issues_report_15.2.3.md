## 设计问题清单

### 问题 1: int 和 number 的关系不明确

**规范要求**:
- ArkTS 规范未明确说明 `int` 和 `number` 的关系
- 测试用例 `SEM_15_02_012_FAIL_INT_SUBTYPE_GAP` 暗示 `int` 不是 `number` 的子类型

**对比分析**:

| 语言 | int 和 number 的关系 | 设计理由 |
|------|---------------------|----------|
| ArkTS | 不明确 | 可能存在设计 GAP |
| Java | `int` 是基本类型，`Integer` 是包装类 | 性能优化，自动装箱/拆箱 |
| Swift | `Int` 是 `Int` 类型，没有单独的 `number` 类型 | 简化类型系统 |

**Java 示例**:
```java
// Java: int 可以赋值给 Integer（自动装箱）
int x = 42;
Integer y = x; // ✅ 自动装箱

// Java: Integer 可以赋值给 int（自动拆箱）
Integer y = 42;
int x = y; // ✅ 自动拆箱
```

**Swift 示例**:
```swift
// Swift: Int 是值类型
let x: Int = 42
let y: Int = x // ✅ 值拷贝
```

**结论（2026-06-22 已确认）**:
- ArkTS 编译器支持 `int` 隐式拓宽到 `number`，这与 Spec 的数值拓宽规则一致（"the minimum type is int"）
- 测试用例 `SEM_15_02_012` 已从 compile-fail 改为 compile-pass，GAP 关闭
- 建议在规范中明确说明 `int → number` 的隐式拓宽关系，减少歧义
2. 考虑让 `int` 成为 `number` 的子类型（类似 TypeScript）
3. 如果不打算支持，需要在规范中明确说明并给出理由

**优先级**: 高

### 问题 2: 字面量类型的表达能力

**规范要求**:
- ArkTS 支持字符串字面量类型
- 未明确说明是否支持数字字面量类型和布尔字面量类型

**对比分析**:

| 语言 | 字符串字面量 | 数字字面量 | 布尔字面量 | 枚举字面量 |
|------|-------------|-----------|-----------|-----------|
| ArkTS | ✅ 支持 | ❓ 未明确 | ❓ 未明确 | ✅ 支持 |
| Java | ❌ 不支持 | ❌ 不支持 | ❌ 不支持 | ✅ 支持（enum） |
| Swift | ❌ 不支持 | ❌ 不支持 | ❌ 不支持 | ✅ 支持（enum） |

**TypeScript 示例** (ArkTS 的参考):
```typescript
// TypeScript: 支持各种字面量类型
type StringLiteral = "hello" | "world";
type NumberLiteral = 1 | 2 | 3;
type BooleanLiteral = true | false;
```

**可能的问题**:
1. ArkTS 的字面量类型表达能力可能不足
2. 未明确说明支持哪些字面量类型，导致用户困惑

**改进建议**:
1. 在规范中明确说明支持哪些字面量类型
2. 考虑增加数字字面量类型和布尔字面量类型的支持
3. 提供参考示例，帮助用户理解字面量类型的使用场景

**优先级**: 中

## 设计规范建议

### 建议 1: 明确字面量类型的子类型规则

**现状**: ArkTS 规范未明确说明字面量类型的子类型规则

**建议**: 在规范中明确说明以下规则：
1. 字符串字面量类型是其基础类型 `string` 的子类型
2. 数字字面量类型是其基础类型 `number` 的子类型（如果支持）
3. 布尔字面量类型是其基础类型 `boolean` 的子类型（如果支持）
4. 枚举字面量类型是其枚举类型的子类型

**示例**:
```arkts
// 建议：明确字面量类型的子类型规则
let literal: "hello" = "hello";
let str: string = literal; // ✅ "hello" <: string

let numLiteral: 42 = 42;
let num: number = numLiteral; // ✅ 42 <: number（如果支持）
```

**优先级**: 高

---

### 建议 2: 增加字面量类型的推断规则

**现状**: ArkTS 规范未明确说明字面量类型的推断规则

**建议**: 在规范中明确说明以下推断规则：
1. 当变量声明时初始化为字面量，推断为字面量类型（而非基础类型）
2. 当变量声明时指定类型为基础类型，可以赋值为字面量

**示例**:
```arkts
// 建议：明确字面量类型的推断规则
let a = "hello"; // 推断为 "hello"，而非 string
let b: string = "hello"; // 推断为 string，可以赋值为 "hello"
```

**优先级**: 中
