ArkTS 采用名义子类型系统（与 Java/Swift 一致），并扩展支持字面量、联合、函数、FixedArray 等子类型规则，这些扩展来自 TypeScript 兼容需求。复杂性与表达能力是固有权衡，非缺陷。

## 设计规范建议

### 建议 1: 详细说明子类型规则

**现状**: ArkTS 规范未详细说明子类型规则

**建议**: 在规范中详细说明以下规则：
1. 自身子类型：`T <: T`
2. 传递性：如果 `A <: B` 且 `B <: C`，则 `A <: C`
3. 类继承：如果 `B extends A`，则 `B <: A`
4. 接口实现：如果 `C implements I`，则 `C <: I`
5. 其他子类型规则（泛型、字面量、元组、联合、函数、FixedArray、交叉类型、差分类型）

**示例**:
```arkts
// 建议：详细说明子类型规则
class Animal {}
class Dog extends Animal {} // Dog <: Animal

let dog: Dog = new Dog();
let animal: Animal = dog; // ✅ Dog <: Animal
```

**优先级**: 高

---
