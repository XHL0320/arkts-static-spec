## 设计规范建议

### 建议 1: 明确元组的子类型规则

**现状**: ArkTS 规范未明确说明元组的子类型规则

**建议**: 在规范中明确说明以下规则：
1. 元组身份子类型：相同结构元组可以互相赋值
2. 元组协变：如果支持，说明协变规则（元素类型协变）
3. 元组逆变：如果支持，说明逆变规则（元素类型逆变）
4. 元组长度：必须匹配（或不匹配时的处理规则）

**示例**:
```arkts
// 建议：明确元组的子类型规则
let t1: [int, string] = [1, "a"];
let t2: [int, string] = t1; // ✅ 身份子类型

// 如果支持协变：
let t3: [number, any] = t1; // ✅ 协变（int <: number, string <: any）
```

**优先级**: 高

---

### 建议 2: 增加元组的类型推断规则

**现状**: ArkTS 规范未明确说明元组的类型推断规则

**建议**: 在规范中明确说明以下推断规则：
1. 当变量声明时初始化为元组字面量，推断为元组类型
2. 当变量声明时指定类型为元组类型，可以赋值为相同结构的元组

**示例**:
```arkts
// 建议：明确元组的类型推断规则
let a = [1, "a"]; // 推断为 [int, string]
let b: [int, string] = [1, "a"]; // ✅ 类型匹配
```

**优先级**: 中
