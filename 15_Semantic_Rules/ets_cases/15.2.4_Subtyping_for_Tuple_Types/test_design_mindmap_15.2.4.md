# 15.2.4 Subtyping for Tuple Types - 测试设计思维导图

## 测试范围

**规范章节**: 15.2.4 Subtyping for Tuple Types

**测试目标**: 验证 ArkTS 中元组类型的子类型关系，包括相同结构元组的赋值、元组元素类型的子类型关系等。

**关键概念**:
- 元组身份子类型: `[T1, T2]` 可以赋值给 `[T1, T2]`（相同结构）
- 元组协变: 如果 `T1 <: U1` 且 `T2 <: U2`，则 `[T1, T2] <: [U1, U2]`（如果支持协变）
- 元组长度: 元组长度必须匹配

## 测试设计思路

### 1. 元组身份子类型（正向）
- **测试点**: 相同结构元组可以赋值给相同结构的元组变量
- **测试代码**: `let t1: [int, string] = [1, "a"]; let t2: [int, string] = t1;`
- **预期结果**: 编译通过，运行时正常

### 2. 元组元素类型不匹配（反向）
- **测试点**: 元组元素类型不匹配时，不能赋值
- **测试代码**: `let t1: [int, string] = [1, "a"]; let t2: [string, int] = t1; // 应报错`
- **预期结果**: 编译失败

### 3. 运行时行为（运行时）
- **测试点**: 验证元组子类型在运行时的行为
- **测试代码**: 创建元组，进行赋值和访问
- **预期结果**: 运行时行为符合预期


## 最新设计要点

从章节思维导图同步的最新测试设计点：

- identical tuple types
- different tuple types (no subtyping)
