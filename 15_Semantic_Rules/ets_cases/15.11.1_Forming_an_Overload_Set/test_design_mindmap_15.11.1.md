# 15.11.1 Forming an Overload Set - 测试设计思维导图

## 概述
本节定义重载集的形成规则：多个同名函数自动构成重载集，编译器根据参数类型、数量、顺序等信息确定最佳匹配。

## 核心规则

### 1. 重载集形成条件
- 同一作用域内存在多个同名函数
- 函数参数类型、数量或顺序不同
- 仅返回类型不同不构成重载

### 2. 重载决策
- 根据调用参数类型匹配最佳函数
- 类型不匹配时编译报错
- 存在歧义时编译报错

### 3. 重载与重写的交互
- 重载发生在同一作用域内
- 重写发生在父子类之间
- 重写不影响重载集形成

## 测试点覆盖

### compile-pass
1. 同名函数构成重载集
2. 参数类型不同的重载
3. 参数数量不同的重载
4. 参数顺序不同的重载

### compile-fail
1. 类型不匹配拒绝
2. 仅返回类型不同的重载拒绝
3. 重载歧义拒绝

### runtime
1. 重载函数运行时正确调用
2. 重载决策运行时验证

## 编号规划
- compile-pass: 001 ~ 010
- compile-fail: 011 ~ 020
- runtime: 021 ~ 030
