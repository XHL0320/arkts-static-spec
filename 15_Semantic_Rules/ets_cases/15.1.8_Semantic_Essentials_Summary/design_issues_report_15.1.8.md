## 二、与业界静态语言差异点 (Differences from Industry Static Languages)

### 2.1 ArkTS 语义基础与 Java 的高度一致性
- **ArkTS 行为**: ArkTS 语义基础与 Java 高度一致（数值拓宽、字符串拼接、赋值规则等）
- **Java 行为**: 作为静态类型语言的基准
- **Swift 行为**: 部分语义不同（如数值拓宽需要显式转换）
- **TypeScript 行为**: 结构类型系统不同，语义基础有差异
- **差异分析**: ArkTS 与 Java 高度一致，与 TypeScript 的结构类型系统不同
- **设计意图**: 保持与 Java 一致的语义基础，降低开发者学习成本

### 2.2 类型推断能力的差异
- **ArkTS 行为**: 类型推断能力有限（类型参数作为 LHS 时不提供推断信息）
- **Java 行为**: 类似，类型推断能力有限
- **Swift 行为**: 支持强大的上下文推断
- **TypeScript 行为**: 支持强大的上下文推断
- **差异分析**: ArkTS 和 Java 的类型推断能力较弱，Swift 和 TypeScript 更强大
- **设计意图**: 保持与 Java 一致的简单类型推断语义
