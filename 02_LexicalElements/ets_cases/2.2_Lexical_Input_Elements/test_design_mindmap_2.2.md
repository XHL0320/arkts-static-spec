# 2.2 Lexical Input Elements - 测试设计思维导图

## 概述
ArkTS 词法输入元素分为四类：White Spaces、Line Separators、Tokens、Comments。本节测试覆盖这四类元素的识别、组合、分隔和边界交互。

## 子类型覆盖

### 1. 四类输入元素共存
- **同行包含全部四类元素**
  - 正向编译: 一行代码中同时存在空白、Token、注释
  - 反向编译: (无)
  - 运行时: 含注释的语句执行结果正确
- **多行混合四类元素**
  - 正向编译: 跨行混合空白、换行、Token、注释
  - 反向编译: (无)
  - 运行时: 混合元素不影响执行语义

### 2. 空白符作为分隔符
- **空白符分隔Token**
  - 正向编译: 空格/Tab/多个空白分隔标识符与关键字
  - 反向编译: 无空白符导致Token合并(如 `letx` 非法)
  - 运行时: 空白分隔不影响语义
- **空白符在运算符间**
  - 正向编译: `a + b`、`a+b`、`a  +  b` 均合法
  - 反向编译: (无)
  - 运行时: 不同空白风格计算结果一致

### 3. 行终止符作为分隔符
- **换行分隔语句**
  - 正向编译: 每行一条语句、行尾可选分号
  - 反向编译: (无)
  - 运行时: 换行分隔的多条语句顺序执行
- **换行在表达式内部**
  - 正向编译: 模板字符串内的换行、括号内换行
  - 反向编译: 字符串字面量内未转义换行
  - 运行时: 多行表达式结果正确

### 4. 注释作为元素
- **单行注释**
  - 正向编译: `//` 注释、行末注释、整行注释
  - 反向编译: (无)
  - 运行时: 注释不影响变量值
- **多行注释**
  - 正向编译: `/* */` 跨行注释、嵌套注释(若支持)
  - 反向编译: 未闭合的多行注释
  - 运行时: 注释内的代码不执行
- **注释与空白交互**
  - 正向编译: 注释替代空白符分隔Token
  - 反向编译: (无)
  - 运行时: 注释替代空白不影响语义

### 5. Token识别
- **Token被空白/换行/注释分隔**
  - 正向编译: 各种分隔组合下的Token识别
  - 反向编译: (无)
  - 运行时: Token识别正确性
- **连续Token无分隔**
  - 正向编译: `a+b`、`foo()` 等运算符/括号无需空白
  - 反向编译: `letx`(标识符合并)、`123abc`(数字与标识符合并)
  - 运行时: 连续Token解析正确

### 6. 边界交互
- **空文件**
  - 正向编译: 仅含空白/换行/注释的文件
  - 反向编译: (无)
  - 运行时: (无)
- **连续行终止符**
  - 正向编译: 多个空行
  - 反向编译: (无)
  - 运行时: 空行不影响后续语句
- **注释内含行终止符**
  - 正向编译: 多行注释内的换行
  - 反向编译: 单行注释后代码延续到下一行
  - 运行时: 多行注释内换行不影响语义

## 分类说明
- **compile-pass**（.ets 文件必须编译成功）
- **compile-fail**（.ets 文件必须产生编译时错误）
- **runtime**（.ets 文件测试运行时行为）

## 文件命名规范
- 前缀: LEX_02_02 表示 Chapter 2 Section 2
- 示例: `LEX_02_02_001_PASS_FOUR_ELEMENT_TYPES.ets`
- 示例: `LEX_02_02_010_FAIL_UNTERMINATED_COMMENT.ets`
