# 2.2 Lexical Input Elements - ArkTS vs Java vs Swift 对比报告

**生成日期：** 2026-06-15
**规范来源：** ArkTS Static Spec §2.2, Java JLS SE21 §3, Swift Language Reference (Lexical Structure)

---

## 一、词法输入元素分类对比

| 输入元素类型 | ArkTS | Java | Swift |
|------------|-------|------|-------|
| White Spaces | ✅ 空格/Tab/FF | ✅ 空格/Tab/FF | ✅ 空格/Tab |
| Line Separators | ✅ LF/CR/CRLF | ✅ LF/CR/CRLF | ✅ LF/CR/CRLF |
| Comments | ✅ // 和 /* */ | ✅ // 和 /* */ | ✅ // 和 /* */ |
| Tokens | ✅ 标识符/关键字/字面量/运算符/分隔符 | ✅ 同左 | ✅ 同左 |

**结论：** 三语言的词法输入元素分类基本一致。

---

## 二、空白符处理对比

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 空白作Token分隔 | ✅ | ✅ | ✅ |
| 运算符间空白可选 | ✅ `a+b` / `a + b` | ✅ | ✅ |
| 多个空白等价一个 | ✅ | ✅ | ✅ |
| Tab 与空格等价 | ✅ | ✅ | ✅ |

**结论：** 三语言完全一致。

---

## 三、行终止符处理对比 ⭐

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 换行作语句分隔 | ✅（需确认ASI） | ❌（必须分号） | ✅（换行即分隔） |
| ASI（自动分号插入） | ⚠️ spec 未明确 | ❌ | ❌（换行天然分隔） |
| 行尾分号 | 可选（待确认） | 必须 | 可选 |
| 字符串内换行 | ❌（需模板字符串） | ❌ | ❌（需多行字面量） |
| 括号内换行 | ✅ | ✅ | ✅ |

**关键差异：** ArkTS 的行终止符语义介于 Java（必须分号）和 Swift（换行即分隔）之间。需通过编译测试确认 ASI 行为。

---

## 四、注释处理对比

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 单行注释 `//` | ✅ | ✅ | ✅ |
| 多行注释 `/* */` | ✅ | ✅ | ✅ |
| 嵌套多行注释 | ⚠️ 待验证 | ❌ | ❌ |
| 注释作Token分隔 | ⚠️ 待验证 | ✅ | ✅ |
| 文档注释 | ✅ `/** */` | ✅ Javadoc | ✅ `///` 和 `/** */` |
| 注释内Unicode | ✅ | ✅ | ✅ |

---

## 五、Token识别对比

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 最长匹配原则 | ✅ | ✅ | ✅ |
| `letx` 解析 | ❌ 编译错误(单标识符) | ❌ | ❌ |
| `a+b` 解析 | ✅ 三个Token | ✅ | ✅ |
| `123abc` 解析 | ❌ 编译错误 | ❌ | ❌ |
| 运算符Token | 多字符: `===`, `!==`, `??`, `?.` | `==`, `!=` | `===`, `!==`, `??` |

**结论：** ArkTS 的运算符Token比 Java 更丰富（受 TypeScript 影响），与 Swift 更接近。

## 六、综合评分

| 维度 | ArkTS | Java | Swift |
|------|-------|------|-------|
| 词法元素分类 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 行终止符设计清晰度 | ⭐⭐⭐（ASI未明确） | ⭐⭐⭐⭐⭐（无ASI） | ⭐⭐⭐⭐⭐（换行分隔） |
| 注释灵活性 | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| Token表达力 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

---

## 七、核心结论

| 角度 | 结论 |
|------|------|
| **分类一致性** | 三语言词法输入元素分类完全一致 |
| **行终止符** | ArkTS 最模糊（ASI 未明确），Java 最明确（必须分号） |
| **Token表达力** | ArkTS 继承 TS 的丰富运算符Token（`===` `??` `?.`），强于 Java |
| **注释** | 三语言基本一致，嵌套注释需实测确认 |
| **建议** | ArkTS spec 应明确 ASI 行为，避免与 TS/JS 的隐式差异 |
