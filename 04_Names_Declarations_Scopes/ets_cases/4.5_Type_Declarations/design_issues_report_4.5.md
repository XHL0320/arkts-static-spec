## 一、行为差异与规范一致性概览

### 问题 A-4.5：递归类型限制策略 ⭐⭐ MEDIUM

**用例：** NAM_04_05_015_FAIL_DIRECT_SELF_REF, NAM_04_05_016_FAIL_UNION_SELF_REF

**实际行为：** ArkTS 禁止直接/间接自引用（如 `type E = E`），但允许部分递归（如 `type A = Array<A>`）。

**Spec 依据：** §4.5 未明确列举所有允许的递归类型模式。

**对比：**
| 语言 | 递归类型支持 |
|------|-------------|
| ArkTS | 部分允许（数组/泛型参数），禁止直接/间接自引用 |
| Java | N/A（无类型别名） |
| Swift | 允许递归类型别名（indirect enum） |
| TypeScript | 允许递归类型 |

**影响：** 限制了某些设计模式（如链表、树等递归数据结构）。

**建议：** Spec 应明确列举所有允许的递归类型模式。
